ImageOps_图片操作 ​

模块简介 ​

ImageOps 封装图片加载、格式转换、缩放/旋转/倾斜/裁剪、模糊处理、水印、圆形/圆角、压缩与保存，以及图片信息查询等功能。

功能概览 ​

  • 加载：路径/File 转 Bitmap，判断是否为图片
  • 格式转换：Bitmap ↔ ByteArray（JPEG/PNG/WEBP/默认）、Bitmap ↔ Mat、Bitmap ↔ Base64
  • 基本处理：缩放（尺寸/比例自动判断）、旋转、倾斜、裁剪、灰度化
  • 模糊处理：快速模糊、RenderScript 模糊、堆栈模糊
  • 水印：文字水印、图片水印
  • 圆形/圆角：转圆形、统一圆角
  • 保存：指定格式保存（路径或相册）
  • 压缩：按比例、按质量、按采样率压缩
  • 信息：获取文件信息 JSON、获取 Bitmap 信息 JSON
  • 相似度比较：均值哈希（aHash）、差值哈希（dHash）、直方图对比
  • 克隆：深拷贝 Bitmap
  • 资源管理：回收 Bitmap 资源

10. 相似度比较 ​

compare() ​

比较两个 Bitmap 图片的相似度。支持多种算法方案。

参数名类型必填说明
bmp1Bitmap?是第一张图片
bmp2Bitmap?是第二张图片
methodString?是比较方法，支持 "avgHash"/"aHash"（均值哈希）、"diffHash"/"dHash"（差值哈希）、"histogram"/"hist"（直方图对比），不区分大小写
返回值类型说明
Float相似度（0.0-1.0），1.0 表示完全相同。出错或参数无效时返回 -1.0

方法说明：

  • avgHash / aHash（均值哈希）：将图片缩放到 8×8 并转为灰度，计算像素均值，每位与均值比较生成 64 位哈希指纹，最后通过汉明距离计算相似度。适合比较整体结构相似的图片。
  • diffHash / dHash（差值哈希）：将图片缩放到 9×8 并转为灰度，逐行比较相邻像素差异生成 64 位哈希指纹，最后通过汉明距离计算相似度。对图片的渐变和亮度变化更敏感。
  • histogram / hist（直方图对比）：将图片缩放到 128×128，统计 RGB 各通道 64 bins 的颜色直方图，通过归一化互相关计算颜色分布相似度。适合比较颜色分布相似的图片。

示例：

lua
local imgOps = UtilsMain.imageOps()
local bmp1 = imgOps.getBitmap("/sdcard/pic1.jpg")
local bmp2 = imgOps.getBitmap("/sdcard/pic2.jpg")

-- 均值哈希比较
local sim1 = imgOps.compare(bmp1, bmp2, "avgHash")
print("均值哈希相似度: " .. sim1)

-- 差值哈希比较
local sim2 = imgOps.compare(bmp1, bmp2, "diffHash")
print("差值哈希相似度: " .. sim2)

-- 直方图比较
local sim3 = imgOps.compare(bmp1, bmp2, "histogram")
print("直方图相似度: " .. sim3)

clone() ​

克隆（深拷贝）一个 Bitmap，生成一个独立的全新副本。

新 Bitmap 与原 Bitmap 具有相同的像素数据和配置（Config、尺寸等），但彼此独立，互不影响。原 Bitmap 回收后克隆的 Bitmap 仍然可用。

参数名类型必填说明
bitmapBitmap?是需要克隆的 Bitmap 对象
返回值类型说明
Bitmap?克隆后的新 Bitmap 对象，失败（如参数为 null、已回收、或内存不足）返回 null

示例：

lua
local imgOps = UtilsMain.imageOps()
local bmp = imgOps.getBitmap("/sdcard/pic.jpg")

-- 克隆一份独立的副本
local clonedBmp = imgOps.clone(bmp)

-- 原图回收后，克隆的图片仍然可用
imgOps.recycle(bmp)
print(imgOps.getInfo(clonedBmp))  -- 仍然可以正常获取信息

imgOps.recycle(clonedBmp)

12. 资源管理 ​

recycle() ​

回收 Bitmap 资源。注意 ：回收后的 Bitmap 不能再 使用，否则会抛出异常。

recycle(bitmap) - 回收单个 Bitmap

参数名类型必填说明
bitmapBitmap?是需要回收的单个 Bitmap 对象
返回值类型说明
void无返回值

示例：

lua
local imgOps = UtilsMain.imageOps()
local bmp1 = imgOps.getBitmap("/sdcard/pic1.jpg")
local bmp2 = imgOps.getBitmap("/sdcard/pic2.jpg")

-- 回收单个 Bitmap
imgOps.recycle(bmp1)
imgOps.recycle(bmp2)

import('com.nx.assist.lua.LuaEngine')
local loader = LuaEngine.loadApk('xfxPlugin-release.apk'); assert(loader)
local context = LuaEngine.getContext()
local UtilsMain = loader.loadClass('com.xfx.plugin.UtilsMain'); assert(UtilsMain)
UtilsMain.init(context)

local imgOps = UtilsMain.imageOps()

-- 加载
local bmp = imgOps.getBitmap("/sdcard/pic.jpg")
local bmp2 = imgOps.getBitmap(java.io.File("/sdcard/pic.jpg"))
print(imgOps.isImage("/sdcard/pic.jpg"))

-- 格式转换
local bytesPng = imgOps.bitmapToBytes(bmp)
local bytesJpg = imgOps.bitmapToBytes(bmp, "JPG", 90)
local bytesWebp = imgOps.bitmapToBytes(bmp, "WEBP", 80)
local fromBytes = imgOps.bytesToBitmap(bytesWebp)

-- Bitmap 与 Mat 转换

local bmpFromMat = imgOps.matToBitmap(mat)

local blurred2 = imgOps.renderScriptBlur(bmp, 25.0)

-- 水印
local watermarked1 = imgOps.addTextWatermark(bmp, "水印", 48, 0xFF000000, 100.0, 100.0)
local watermarkBmp = imgOps.getBitmap("/sdcard/watermark.png")
local watermarked2 = imgOps.addImageWatermark(bmp, watermarkBmp, 50, 50, 200)

-- 圆形与圆角
local round = imgOps.toRound(bmp)
local corner = imgOps.toRound(bmp, 20)

-- 保存
imgOps.save(bmp, "/sdcard/out.jpg", "JPG", 90)
imgOps.save(bmp, "/sdcard/out.png", "PNG", 100)
local albumPath = imgOps.saveToAlbum(bmp, "JPG", 90)

-- 压缩
local compressed1 = imgOps.compressByScale(bmp, 0.5, 0.5)
local compressed2 = imgOps.compressByQuality(bmp, 80)
local compressed3 = imgOps.compressBySampleSize(bmp, 2)

-- 信息
print(imgOps.getInfo("/sdcard/out.jpg"))
local bmpInfo = imgOps.getInfo(bmp)
print(bmpInfo)

-- 相似度比较
local simAvg = imgOps.compare(bmp, bmp2, "avgHash")
print("均值哈希相似度: " .. simAvg)
local simDiff = imgOps.compare(bmp, bmp2, "diffHash")
print("差值哈希相似度: " .. simDiff)
local simHist = imgOps.compare(bmp, bmp2, "histogram")
print("直方图相似度: " .. simHist)

-- 克隆
local clonedBmp = imgOps.clone(bmp)
print("克隆后图片信息: " .. imgOps.getInfo(clonedBmp))

-- 资源管理
imgOps.recycle(bmp)
imgOps.recycle(bmp2)
imgOps.recycle(scaled)
imgOps.recycle(rotated)
imgOps.recycle(clonedBmp)

OOP例子 ​

某游戏，画面有轻微变化，测试不同算法的图片相似度。

lua
local xfxModule = require('lib/XfxPlugin')
local XFX = xfxModule:new({apkName = 'xfxPlugin-release.apk'})

local imgOps = XFX:getOps('imageOps'); assert(imgOps)

for i = 1, 10 do
	local bmp = LuaEngine.snapShot(0, 0,0,0)
	sleep(1000)
	local bmp2 = LuaEngine.snapShot(0, 0,0,0)

	-- 相似度比较
	local simAvg = imgOps.compare(bmp, bmp2, "avgHash")
	print("均值哈希相似度: " .. simAvg)
	local simDiff = imgOps.compare(bmp, bmp2, "diffHash")
	print("差值哈希相似度: " .. simDiff)
	local simHist = imgOps.compare(bmp, bmp2, "histogram")
	print("直方图相似度: " .. simHist)

	-- 资源管理
	imgOps.recycle(bmp)
	imgOps.recycle(bmp2)

	sleep(1000)
end

注意事项 ​

  • 内存：Bitmap 处理可能占用大量内存，处理完成后建议使用 recycle() 回收资源以免出现内存泄露。
  • 路径与权限：读取/保存文件需存储权限；路径需可访问。
  • 格式支持：支持的图片格式包括 JPEG、PNG、WEBP。
  • 缩放参数：scale() 方法会自动判断参数是比例（0-1）还是绝对尺寸（>1），使用时要确保参数类型正确。
  • Base64 转换：bitmapToBase64() 返回纯 Base64 字符串（不含 data URI 前缀），base64ToBitmap() 会自动处理可能存在的 data URI 前缀。
