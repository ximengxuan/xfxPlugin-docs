FloatLog_悬浮窗日志 ​

模块简介 ​

FloatLog 提供悬浮日志窗口：创建/显示/移动/缩放/最小化、日志添加导出、计时器与进度显示、Toast 弹窗等。

⚠️ 不支持懒人精灵最高权限下运行(最高权限下获取不到Context)。

功能概览 ​

  • 上下文设置：设置 Context（setContext，与 UtilsMain.init 二选一）
  • 窗口管理：显示、移动、缩放、最小化/恢复、完全显示/隐藏、透明度、圆角、状态导出/关闭
  • 日志操作：添加日志（多重样式）、清空、获取 JSON、导出文件、最大条数限制
  • 进度与标题：设置标题、进度值/颜色、信息文本
  • 控制回调：设置停止回调（setOnStopCallback）

显示效果 ​

悬浮窗日志模块，由计时器、任务标题区、缩放按钮、日志内容区、进度条、底部栏、关闭按钮等部分组成。每个部分均可自定义显示/隐藏，支持拖动、缩放、最小化/恢复等操作。

API 接口列表 ​

1. 窗口管理 ​

setContext() ​

按需设置 Context（建议传入 applicationContext），在未调用 UtilsMain.init 时也可提前注入。

setContext()和 UtilsMain.init() 二选一调用即可。

参数名类型必填说明
contextContext是应用或 Activity 的 context，建议 applicationContext
返回值类型说明
void无

示例：

lua
import('com.nx.assist.lua.LuaEngine')

local floatLogOps = UtilsMain.floatLog()

local context = LuaEngine.getContext()
floatLogOps.setContext(context)  -- 传入已有的 context

show() ​

在UI线程创建并异步显示悬浮窗口。

必须先设置上下文，才能显示。

参数名类型必填说明
xInt是初始 X（px）
yInt是初始 Y（px）
wInt是初始宽（px）
hInt是初始高（px）
topTextSizeFloat是顶部计时器和标题文字大小 sp
bottomTextSizeFloat是底部信息栏文字大小 sp
isShowTimerBoolean是是否显示左上角计时器
isShowTitleBoolean是是否显示任务信息标题
isShowMinBtnBoolean是是否显示右上角最小化按钮
isShowProgressBarBoolean是是否显示进度条
isShowBottomBarBoolean是是否显示底部信息栏
isShowControlBtnBoolean是是否显示右下角控制按钮
isAllowDragBoolean是是否允许标题栏拖动
isSaveDragPositionBoolean是是否保存拖动位置(左边和顶边)
isSaveSizeBoolean是是否保存窗口尺寸(宽度和高度)
返回值类型说明
void无

示例：

lua
local floatLogOps = UtilsMain.floatLog()
floatLogOps.show(100, 100, 800, 600, 12, 12, true, true, true, true, true, true, true, true, true)

lua
import('com.nx.assist.lua.LuaEngine')

local loader = LuaEngine.loadApk('xfxPlugin-release.apk')
local UtilsMain = loader.loadClass('com.xfx.plugin.UtilsMain')

local floatLogOps = UtilsMain.floatLog()

floatLogOps.toggleVisible()  -- 显示

说明：

  • setVisible(false): 完全隐藏窗口，包括背景和所有内容，不占用屏幕空间
  • setVisible(true): 重新显示之前隐藏的窗口

注意事项 ​

  • 初始化要求：使用前需先调用 xxxOps.setContext(context)或UtilsMain.init(context) 初始化上下文。
  • 线程约束：所有方法在插件内部已强制在UI线程异步执行，无需在调用时，再次设置UI线程执行；
  • 权限：悬浮窗与 Toast 需系统悬浮窗权限（TYPE_APPLICATION_OVERLAY）；存储导出需读写权限。
  • 性能：日志量大时注意最大条数设置；
  • 画面遮挡：悬浮窗日志模块在屏幕上会遮挡下层的内容，并且不能穿透点击。在实际使用的时候，注意设置合适的位置和大小，避免遮挡重要内容导致无法识别（纯节点脚本无需考虑遮挡问题）。
