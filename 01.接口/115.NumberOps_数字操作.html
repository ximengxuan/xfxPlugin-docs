NumberOps_数字操作 ​

模块简介 ​

NumberOps 是数字操作工具模块，封装了常用的数字处理功能，包括数字格式化、类型转换、随机数生成、概率触发等。

功能概览 ​

  • 数字格式化：支持 Float 和 Double 类型的多种格式化方式，可控制小数位数、整数位数、千分位分隔符等
  • 类型转换：Float 转 Double
  • 随机数生成：生成指定范围内的随机整数或小数
  • 概率触发：基于概率值判断是否触发某个事件

3. 随机数生成类 ​

random() ​

生成随机小数或整数。

规则如下：

  • 当 min 和 max 都是整数值时，返回范围内随机整数（包含 min 和 max）
  • 当 min 或 max 为浮点数时，返回随机浮点数，并按 decimals 保留小数位
参数名类型必填说明
minDouble是最小值（包含），必须小于等于 max
maxDouble是最大值（包含），必须大于等于 min
decimalsInt否保留小数位数（仅浮点模式生效），默认 2
返回值类型说明
Number整数模式返回 Int/Long；小数模式返回 Double

注意事项：

  • 如果 max < min，会抛出 IllegalArgumentException 异常
  • 如果 decimals < 0，会抛出 IllegalArgumentException 异常
  • 小数模式会进行四舍五入

示例：

lua
local numOps = UtilsMain.numberOps()

local intRand = numOps.random(1, 10)
print(intRand)  -- 输出: 例如 7（整数，范围含 1 和 10）

local floatRand1 = numOps.random(1.5, 3.5)
print(floatRand1)  -- 输出: 例如 2.37（默认保留 2 位小数）

local floatRand2 = numOps.random(0.1, 0.9, 4)
print(floatRand2)  -- 输出: 例如 0.4567（保留 4 位小数）

randomDecimal() ​

生成指定范围内的随机小数，保留指定小数位数（四舍五入）。

参数名类型必填说明
minDouble是最小值（包含），必须小于等于 max

返回值类型说明
Double随机生成的小数

  • 如果 max < min，会抛出 IllegalArgumentException 异常
  • 如果 decimalPlaces < 0，会抛出 IllegalArgumentException 异常
  • 结果会进行四舍五入到指定小数位数

示例：

lua
local numOps = UtilsMain.numberOps()
local result1 = numOps.randomDecimal(1.0, 10.0, 2)

local result2 = numOps.randomDecimal(0.0, 1.0, 4)
print(result2)  -- 输出: 例如 0.3456 (0.0 到 1.0 之间的随机小数，保留4位)

生成指定范围内的随机整数（包含最小值和最大值，可为负数）。

4. 概率触发类 ​

probabilityTrigger() ​

根据给定的概率值（0-100）判断是否触发某个事件。

参数名类型必填说明
probabilityInt是概率值（0-100），表示触发概率的百分比
返回值类型说明
Booleantrue 表示触发，false 表示不触发

注意事项：

  • 如果 probability < 0，会被限制为 0，返回 false
  • 如果 probability > 100，会被限制为 100，返回 true
  • 如果 probability == 0，直接返回 false
  • 如果 probability == 100，直接返回 true
  • 其他情况下，随机生成 0-100 的整数，如果随机值 <= probability，返回 true

示例：

lua
local numOps = UtilsMain.numberOps()
local result1 = numOps.probabilityTrigger(0)
print(result1)  -- 输出: false (0% 概率，永远不触发)

local result2 = numOps.probabilityTrigger(100)
print(result2)  -- 输出: true (100% 概率，永远触发)

local result3 = numOps.probabilityTrigger(50)
print(result3)  -- 输出: true 或 false (50% 概率，随机触发)

local result4 = numOps.probabilityTrigger(30)
print(result4)  -- 输出: true 或 false (30% 概率，随机触发)

-- 测试多次，观察触发频率
local count = 0
for i = 1, 1000 do
    if numOps.probabilityTrigger(30) then
        count = count + 1
    end
end
print("触发次数: " .. count)  -- 输出: 大约 300 次左右（30% 概率）

-- ============================================
-- 数字操作示例
-- 演示如何使用 XfxPlugin.lua 调用 numOps 进行数字处理

print('格式化 99.9 (2位小数，保留尾随零): ' .. formatted2)

-- ============================================
-- 2. 随机整数生成
-- ============================================
print('\n--- 2. 随机整数生成 ---')

for i = 1, 5 do
    local random = numOps.randomInt(1, 100)

-- ============================================
-- 3. 随机小数生成

-- 4. 概率触发

print('\n--- 4. 概率触发 ---')

local probability = 30  -- 30% 概率
local successCount = 0
for i = 1, 100 do
    if numOps.probabilityTrigger(probability) then
        successCount = successCount + 1
    end
end
print('100 次测试，30% 概率，成功次数: ' .. tostring(successCount))

print('\n========== 数字操作示例完成 ==========')

注意事项 ​

    • randomDecimal() 在 decimalPlaces < 0 时会抛出 IllegalArgumentException 异常
    • 调用前请确保参数合法性

  • 格式化精度：数字格式化方法会根据指定的小数位数进行四舍五入处理。

  • 随机数范围：randomInt() 和 randomDecimal() 都包含最小值和最大值本身，即结果可能等于 min 或 max。
