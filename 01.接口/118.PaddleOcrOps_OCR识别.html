PaddleOcrOps_OCR识别 ​

模块简介 ​

PaddleOcrOps 提供 PaddleOCR 文字识别接口，适合安卓移动端截图、局部图片、游戏 UI 文本识别等场景。

已接入 NCNN 推理引擎，OCR 默认使用 CPU 推理，优先保证识别稳定性。

模型要求 ​

插件已内置 PP-OCR v4 / v5 / v6 三套 PP-OCR mobile NCNN 模型。

调用 init() 默认加载 v6； 也可通过 init(version) 指定版本、initFromDir(modelDir) 按目录自动发现模型，或通过带 5 个文件路径的 init(...) 加载自定义模型。

版本assets 目录释放目录名说明
v4ppocrv4xfx_ppocrv4旧版 CRNN 识别链路，32 高输入
v5ppocrv5xfx_ppocrv5官方 NCNN mobile 模型
v6ppocrv6xfx_ppocrv6默认版本，识别精度更高

version 参数忽略大小写，支持 v4、V5、6 等写法；传入不支持的版本会返回 1001 参数错误。

自定义模型需要提供以下文件：

文件说明
det.ncnn.param文本检测模型结构
det.ncnn.bin文本检测模型权重
rec.ncnn.param文字识别模型结构
rec.ncnn.bin文字识别模型权重
keys.txt文字字典文件，每行一个字符

推荐优先使用 mobile 模型(server 模型体积更大，初始化和识别耗时更高)。

自定义模型需要与当前 PaddleOCR/NCNN 预处理和 CTC 解码链路匹配。

PP-OCRv4 这类旧 CRNN 识别模型会自动使用 32 高识别输入和 FP32 识别分支；

如果仍返回文字乱码，请优先确认 rec.param、rec.bin 与 keys.txt 同源，并确认模型不依赖宿主 APK 私有的额外预处理或后处理。

功能概览 ​

  • 多版本内置模型：内置 v4 / v5 / v6，默认 v6，可通过 init(version) 切换
  • 目录自动发现：initFromDir(modelDir) 自动匹配 det/rec/keys 共 5 个文件
  • 自定义模型：加载外部检测模型、识别模型和字典
  • OCR 识别：对 Bitmap 执行文字检测与识别，返回 JSON 字符串
  • 阈值控制：支持检测阈值、文本框扩展比例、识别分数阈值、识别输入最大宽度
  • 性能优化：识别阶段并行执行，默认限制识别输入最大宽度为 320
  • 状态查询：初始化状态、native 版本、最近错误码和错误消息
  • 资源管理：释放 native 模型资源，适配脚本结束、Activity 销毁、切换模型等场景

init(...) 自定义模型（显式路径） ​

加载自定义 NCNN 模型，手动指定全部文件路径。

参数名类型必填说明
detParamPathString是文本检测模型 param 文件路径

print(ok)
print(PaddleOps.getLastErrorCode())

目录自动发现规则（仅扫描该目录下文件，不递归子目录）：

角色匹配方式
det param / bin文件名分段含 det，扩展名为 .param / .bin（忽略 cls）
rec param / bin文件名分段含 rec，扩展名为 .param / .bin（忽略 cls）
keys优先 keys.txt，否则文件名含 keys 的 .txt

支持常见命名，例如：

  • PP_OCRv5_mobile_det.ncnn.param + PP_OCRv5_mobile_det.ncnn.bin
  • det.param + det.bin + rec.param + rec.bin + keys.txt

OCR 识别 ​

detect() ​

使用默认阈值执行 OCR 识别。

范围识别比全屏识别要更准确，速度也更快。大多数场景下，建议采用范围识别。

参数名类型必填说明
bitmapBitmap是待识别 Bitmap，不能为 nil 或已回收
lua
local bmp = LuaEngine.snapShot(0, 0, 280, 271)
local resultJson = PaddleOps.detect(bmp)
LuaEngine.releaseBmp(bmp)
print(resultJson)

detect() ​

使用自定义阈值执行 OCR 识别。

参数名类型必填说明
bitmapBitmap是待识别 Bitmap
boxThresholdFloat是文本检测阈值，范围 0.0 到 1.0，默认 0.30
unclipRatioFloat是文本框扩展比例，必须大于 0，默认 1.60
recScoreThresholdFloat是识别分数阈值，范围 0.0 到 1.0，默认 0.0；用于过滤score低分结果
maxRecWidthInt否识别模型输入最大宽度，默认 320，范围 8 到 2048；越小识别速度越快，长文本精度可能下降

maxRecWidth 默认 320，适合移动端速度优先；长文本识别不完整时可尝试 480 或 640。

lua
local resultJson = PaddleOps.detect(bmp, 0.30, 1.60, 0.6)
local hpJson = PaddleOps.detect(bmp, 0.5, 1.0, 0.8, 80)   -- 竖屏：快速识别
local spJson = PaddleOps.detect(bmp, 0.3, 1.0, 0.5, 120)   -- 横屏：快速识别

错误码 ​

code说明常见原因
0成功正常
1001参数错误空路径、Bitmap 无效、阈值超出范围、不支持的模型版本
1002文件不存在param、bin 或 keys 路径错误，或默认模型释放失败
2001native 库加载失败APK 未包含 libxfx_yolo.so 或 ABI 不匹配
2002native 初始化失败模型格式不匹配、字典为空、native 返回空句柄
3001未初始化调用 detect 前没有成功 init
3002识别失败Bitmap 格式不支持、native 推理异常

lua
import('com.nx.assist.lua.LuaEngine')

-- 加载 XfxPlugin.lua 类
local xfxModule = require('lib/XfxPlugin')

-- 创建 XFX 对象实例
local XFX = xfxModule:new({
    apkName = 'xfxPlugin-release.apk',  -- APK 文件名
    context = LuaEngine.getContext(),  -- 上下文对象
})

-- 获取屏幕信息
local screenOps = XFX:getOps('screenOps')
local screenWidth = screenOps.getScreenWidth()
local screenHeight = screenOps.getScreenHeight()
print('屏幕宽度: ' .. tostring(screenWidth))
print('屏幕高度: ' .. tostring(screenHeight))

-- 打开浮窗日志，方便观察 OCR 识别状态
XFX:showFloatLogWindow({
    left = math.floor(screenWidth / 2),
    top = math.floor(screenHeight / 4),
    width = math.floor(screenWidth / 2),
    height = math.floor(screenHeight / 6),
    isShowProgressBar = true,
    isBottomInfo = true,
    isControlBtn = true,
})

XFX:setFloatLogTitle('XFX PaddleOCR Demo')
XFX:setFloatLogInfo('准备初始化 OCR...')

-- 获取 PaddleOps 对象
local PaddleOps = XFX:getOps('paddleOcrOps'); assert(PaddleOps)

-- 初始化（使用内置默认模型）
local ok = PaddleOps.init()

print(ok)
print(PaddleOps.getLastErrorCode())
print(PaddleOps.getLastErrorMessage())
-- print(PaddleOps.getDefaultModelDir())

if not ok then
  XFX:loge('OCR 初始化失败: ' .. tostring(PaddleOps.getLastErrorMessage()))
  XFX:setFloatLogInfo('OCR 初始化失败')
else
  XFX:logs('OCR 初始化成功')
  XFX:setFloatLogInfo('OCR 初始化成功，开始识别...')

  -- 截图识别
  local bmp = LuaEngine.snapShot(0, 0, 0, 0)
  local resultJson = PaddleOps.detect(bmp, 0.30, 1.0, 0.5, 120)
  bmp.recycle()
  XFX:logi('识别结果: ' .. tostring(resultJson))
end

-- 释放资源
PaddleOps.release()
XFX:logs('OCR 资源已释放')

sleep(5000)

XFX:closeFloatLog()
