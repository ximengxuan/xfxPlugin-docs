## 模块简介

`CosOps` 是腾讯云对象存储（COS）的操作封装，基于 [COS V5 Android SDK](https://cloud.tencent.com/document/product/436/12159) 实现。

支持上传、下载、删除文件，创建、查询、删除存储桶等常用操作。

## 功能概览

- **初始化配置**：使用密钥初始化 COS 服务
- **文件操作**：上传、下载、删除 COS 对象
- **存储桶操作**：创建、查询是否存在、删除存储桶
- **资源释放**：释放 COS 服务资源，避免内存泄漏

## API 接口列表

### 1. 初始化

#### init()

使用永久密钥初始化 COS 服务（仅供测试/开发使用）。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| secretId | String | 是 | SecretId |
| secretKey | String | 是 | SecretKey |
| region | String | 是 | 存储桶地域简称，如 `ap-guangzhou`、`ap-beijing` 等 |
| useHttps | Boolean | 否 | 是否使用 HTTPS，默认为 true |

**地域简称参考：**

| 地域 | 简称 |
|------|------|
| 北京 | ap-beijing |
| 上海 | ap-shanghai |
| 广州 | ap-guangzhou |
|成都 | ap-chengdu |
| 重庆 | ap-chongqing |
| 香港 | ap-hongkong |
| 新加坡 | ap-singapore |
| 东京 | ap-tokyo |

**示例：**
```lua
local cosOps = UtilsMain.cosOps()
cosOps.init("AKIDxxxxxxxxxxxx", "xxxxxxxxxxxxxxxx", "ap-guangzhou", true)
```

#### init() - 临时密钥（生产环境推荐）

使用临时密钥初始化 COS 服务（生产环境唯一安全方式）。

::: warning 安全说明
生产环境必须使用临时密钥方式！临时密钥会自动过期，即使泄露也不会造成严重后果。
临时密钥需要从你的服务器获取，参考 [移动应用直传实践](https://cloud.tencent.com/document/product/436/17025)。
:::

|| 参数名 | 类型 | 必填 | 说明 |
||--------|------|------|------|
|| region | String | 是 | 存储桶地域简称，如 `ap-guangzhou`、`ap-beijing` 等 |
|| tmpSecretId | String | 是 | 临时密钥 SecretId（从你的服务器获取） |
|| tmpSecretKey | String | 是 | 临时密钥 SecretKey |
|| sessionToken | String | 是 | 临时会话令牌 |
|| expiredTime | Long | 是 | 临时密钥过期时间戳（秒） |
|| useHttps | Boolean | 否 | 是否使用 HTTPS，默认为 true |

**示例：**
```lua
local cosOps = UtilsMain.cosOps()

-- 假设你从服务器获取了临时密钥
local serverResponse = httpLib.get("https://your-server.com/get-cos-credentials")
local credentials = jsonLib.parse(serverResponse)

-- 使用临时密钥初始化
cosOps.init(
    credentials.tmpSecretId,           -- 临时 SecretId
    credentials.tmpSecretKey,           -- 临时 SecretKey
    credentials.sessionToken,           -- 会话令牌
    credentials.expiredTime             -- 过期时间戳
)
```

#### isInitialized()

检查 COS 服务是否已初始化。

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 已初始化返回 true |

**示例：**
```lua
local cosOps = UtilsMain.cosOps()
if not cosOps.isInitialized() then
    print("请先初始化 COS 服务")
end
```

#### release()

释放 COS 服务资源，避免内存泄漏。

::: tip 使用建议
在 Activity/Fragment 销毁时调用，例如在 `onDestroy()` 中调用。
调用后需要重新调用 `init()` 才能继续使用。
:::

**示例：**
```lua
local cosOps = UtilsMain.cosOps()

-- 释放资源
cosOps.release()
print("COS 服务已释放")
```

### 2. 文件操作

#### uploadFile()

上传本地文件到 COS（支持大文件分片上传和断点续传）。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bucket | String | 是 | 存储桶名称，格式为 `bucketname-appid` |
| cosPath | String | 是 | 对象在存储桶中的位置标识符（对象键） |
| localFilePath | String | 是 | 本地文件的绝对路径 |
| timeoutSeconds | Long | 否 | 超时时间（秒），默认为 3600（1小时） |

| 返回值类型 | 说明 |
|-----------|------|
| String | 结果 JSON，格式：`{"success": true, "etag": "xxx", "fileSize": 1234567}` |

**文件大小限制：**
- 小于 2MB：简单上传
- 大于等于 2MB：自动使用分片上传，支持断点续传

**示例：**
```lua
local cosOps = UtilsMain.cosOps()

-- 上传文件（使用默认超时 1 小时）
local result = cosOps.uploadFile(
    "mybucket-1250000000",           -- 存储桶名称
    "images/photo.jpg",              -- COS 对象路径
    "/sdcard/Pictures/photo.jpg"     -- 本地文件路径
)
print(result)

-- 上传大文件（设置超时 2 小时）
local result = cosOps.uploadFile(
    "mybucket-1250000000",
    "videos/movie.mp4",
    "/sdcard/Movies/movie.mp4",
    7200  -- 超时 2 小时
)

-- 解析结果
local json = jsonLib.parse(result)
if json.success then
    print("上传成功，文件大小: " .. json.fileSize)
else
    print("上传失败: " .. json.error)
end
```

#### downloadFile()

下载 COS 文件到本地（支持大文件断点续传）。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bucket | String | 是 | 存储桶名称，格式为 `bucketname-appid` |
| cosPath | String | 是 | 对象在存储桶中的位置标识符（对象键） |
| saveDir | String | 是 | 本地保存目录 |
| saveName | String | 否 | 本地保存文件名，为 nil 时使用 COS 上的文件名 |
| timeoutSeconds | Long | 否 | 超时时间（秒），默认为 3600（1小时） |

| 返回值类型 | 说明 |
|-----------|------|
| String | 结果 JSON，格式：`{"success": true, "filePath": "/path/to/file", "fileSize": 1234567}` |

**示例：**
```lua
local cosOps = UtilsMain.cosOps()

-- 下载文件（使用默认超时 1 小时）
local result = cosOps.downloadFile(
    "mybucket-1250000000",           -- 存储桶名称
    "images/photo.jpg",              -- COS 对象路径
    "/sdcard/Download",              -- 保存目录
    "downloaded.jpg"                 -- 保存文件名（可选）
)
print(result)

-- 下载大文件（设置超时 2 小时）
local result = cosOps.downloadFile(
    "mybucket-1250000000",
    "videos/movie.mp4",
    "/sdcard/Movies",
    "downloaded.mp4",
    7200  -- 超时 2 小时
)

-- 解析结果
local json = jsonLib.parse(result)
if json.success then
    print("下载成功，文件路径: " .. json.filePath .. "，大小: " .. json.fileSize)
else
    print("下载失败: " .. json.error)
end
```

#### deleteFile()

删除 COS 文件。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bucket | String | 是 | 存储桶名称，格式为 `bucketname-appid` |
| cosPath | String | 是 | 对象在存储桶中的位置标识符（对象键） |

| 返回值类型 | 说明 |
|-----------|------|
| String | 结果 JSON，格式：`{"success": true}` 或 `{"success": false, "error": "xxx"}` |

**示例：**
```lua
local cosOps = UtilsMain.cosOps()

local result = cosOps.deleteFile(
    "mybucket-1250000000",
    "images/photo.jpg"
)
print(result)

local json = jsonLib.parse(result)
if json.success then
    print("删除成功")
else
    print("删除失败: " .. json.error)
end
```

### 3. 存储桶操作

#### createBucket()

创建新的存储桶。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bucket | String | 是 | 存储桶名称，格式为 `bucketname-appid` |
| region | String | 是 | 存储桶地域简称 |

| 返回值类型 | 说明 |
|-----------|------|
| String | 结果 JSON，格式：`{"success": true, "location": "xxx.cos.ap-guangzhou.myqcloud.com"}` |

**示例：**
```lua
local cosOps = UtilsMain.cosOps()

local result = cosOps.createBucket(
    "mynewbucket-1250000000",
    "ap-guangzhou"
)
print(result)
```

#### bucketExists()

判断存储桶是否存在。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bucket | String | 是 | 存储桶名称，格式为 `bucketname-appid` |

| 返回值类型 | 说明 |
|-----------|------|
| String | 结果 JSON，格式：`{"exists": true}` 或 `{"exists": false}` |

**示例：**
```lua
local cosOps = UtilsMain.cosOps()

local result = cosOps.bucketExists("mybucket-1250000000")
local json = jsonLib.parse(result)
if json.exists then
    print("存储桶存在")
else
    print("存储桶不存在")
end
```

#### deleteBucket()

删除存储桶。

::: warning 注意
存储桶必须为空才能删除成功，请先删除桶内的所有文件。
:::

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bucket | String | 是 | 存储桶名称，格式为 `bucketname-appid` |

| 返回值类型 | 说明 |
|-----------|------|
| String | 结果 JSON，格式：`{"success": true}` 或 `{"success": false, "error": "xxx"}` |

**示例：**
```lua
local cosOps = UtilsMain.cosOps()

local result = cosOps.deleteBucket("myoldbucket-1250000000")
local json = jsonLib.parse(result)
if json.success then
    print("删除成功")
else
    print("删除失败: " .. json.error)
end
```

### 通用操作示例

```lua
local bucket = "mybucket-1250000000"
local localFile = "/sdcard/Pictures/test.jpg"

-- 2. 检查存储桶是否存在
print("检查存储桶...")
local existsResult = cosOps.bucketExists(bucket)
local existsJson = jsonLib.parse(existsResult)

if not existsJson.exists then
    -- 3. 创建存储桶（如果不存在）
    print("创建存储桶...")
    local createResult = cosOps.createBucket(bucket, "ap-guangzhou")
    print("创建结果: " .. createResult)
end

-- 4. 上传文件
if File(localFile).exists() then
    print("上传文件...")
    local uploadResult = cosOps.uploadFile(bucket, "uploads/test.jpg", localFile)
    print("上传结果: " .. uploadResult)
    
    local uploadJson = jsonLib.parse(uploadResult)
    if uploadJson.success then
        -- 5. 下载文件
        print("下载文件...")
        local downloadResult = cosOps.downloadFile(
            bucket, 
            "uploads/test.jpg", 
            "/sdcard/Download", 
            "downloaded.jpg"
        )
        print("下载结果: " .. downloadResult)
        
        -- 6. 删除文件
        print("删除文件...")
        local deleteResult = cosOps.deleteFile(bucket, "uploads/test.jpg")
        print("删除结果: " .. deleteResult)
    end
else
    print("本地文件不存在: " .. localFile)
end

-- 7. 删除存储桶（谨慎操作！）

-- local deleteBucketResult = cosOps.deleteBucket(bucket)
-- print("删除存储桶结果: " .. deleteBucketResult)

print("演示完成！")
```

## 常见问题

### 1. 初始化失败

- **检查密钥**：确保 SecretId 和 SecretKey 正确
- **检查网络**：确保设备可以访问腾讯云服务器

### 2. 上传/下载失败

- **文件不存在**：检查本地文件路径是否正确
- **存储桶不存在**：先创建存储桶
- **权限不足**：确保密钥有对应的读写权限
- **文件过大**：大文件建议使用分片上传

### 3. 删除存储桶失败

- **存储桶非空**：先删除桶内所有文件
- **权限不足**：确保密钥有删除存储桶的权限

### 4. 如何获取密钥和 AppId

[获取 SecretId 和 SecretKey](https://console.cloud.tencent.com/cam/capi)

登录 [腾讯云控制台](https://console.cloud.tencent.com/)，在【对象存储】中可以查看存储桶名称，格式为 `bucketname-appid`，其中数字部分就是 AppId。

### 5. 如何获取临时密钥

临时密钥需要通过腾讯云 STS（Security Token Service）获取，推荐方案：

1. **前端请求后端**：你的 APP/小程序向你的服务器请求临时密钥
2. **后端生成密钥**：服务器使用 CAM 角色或子账号调用 STS 获取临时凭证
3. **返回给前端**：后端将 `tmpSecretId`、`tmpSecretKey`、`sessionToken`、`expiredTime` 返回给客户端

**后端示例（Python）**：
```python
import json
from tencentcloud.common import Credential
from tencentcloud.sts.v20180813 import StsClient

# 使用 CAM 角色或子账号密钥
cred = Credential(secret_id, secret_key)
client = StsClient(cred, "ap-guangzhou")

# 调用 STS 获取临时凭证
response = client.get_credential()
result = {
    "tmpSecretId": response.credentials.tmpSecretId,
    "tmpSecretKey": response.credentials.tmpSecretKey,
    "sessionToken": response.credentials.sessionToken,
    "expiredTime": response.expiredTime
}
return json.dumps(result)
```

- [STS 临时凭证](https://cloud.tencent.com/document/product/1334/49816)

## 注意事项

- **线程安全**：所有操作均为异步执行，不会阻塞主线程
- **网络要求**：需要网络连接才能操作 COS
- **存储桶命名**：存储桶名称全局唯一，建议使用有意义的命名
- **对象键**：COS 中的对象键类似于文件路径，支持 `/` 分隔
- **密钥安全**：生产环境建议使用临时密钥，定期更换密钥
- **资源释放**：不再使用 COS 服务时，务必调用 `release()` 释放资源，避免内存泄漏
