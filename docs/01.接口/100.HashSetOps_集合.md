## 模块简介

`HashSetOps` 提供线程安全的集合（Set）管理，支持按 ID 持有多个集合实例，覆盖创建、添加/移除、包含判断、状态查询、批量导入、清空/删除以及导出 JSON。

除默认的 **ConcurrentHashMap.newKeySet()**（无序、高并发）外，还支持 **LinkedHashSet** 模式：按插入顺序遍历，并可设置最大缓存条数，超出后自动删除最早插入的数据。同时支持通过 **MMKV** 或 **SharedPreferences** 将数据持久化到本地。

典型场景：记录已使用过的 IP 地址（换 IP 去重）、脚本重启后恢复历史 IP 记录、带容量上限的有序去重集合。

## 功能概览

- **实例管理**：创建/删除集合实例，支持 HashSet 与 LinkedHashSet 两种模式
- **容量控制**：LinkedHashSet 模式可设 `maxSize`，超出自动淘汰最早数据
- **持久化**：create 时自动持久化，或手动调用 save/load、saveToMmkv/saveToSp
- **元素操作**：添加、移除、包含判断
- **状态查询**：获取大小、最大条数、是否有序、判断是否为空
- **批量与导出**：批量添加（JSON 数组）、转为 JSON 数组字符串

## API 接口列表

### 1. 实例管理

#### create()

创建或获取集合实例。支持两种调用方式：

**方式一：简单创建（ConcurrentHashMap.newKeySet()，默认）**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |

**方式二：高级创建（LinkedHashSet / 容量限制 / 持久化）**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |
| ordered | Boolean | 是 | 是否使用 LinkedHashSet（按插入顺序遍历） |
| maxSize | Int | 是 | 最大条数，超出后删除最早数据；0 表示不限制 |
| persistType | String | 是 | 持久化类型：`none` / `mmkv` / `sp` |
| persistKey | String | 否 | 持久化存储键，整份集合存为 JSON 数组 |
| storageName | String | 否 | MMKV 实例 ID 或 SP 文件名，空串用默认实例 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 创建/获取成功返回 true |

> **说明**：若实例已存在，直接返回 true，不会覆盖已有配置。`maxSize > 0` 时会自动启用有序模式。启用持久化后，create 时自动从本地加载；add/remove/clear/addAll 后自动保存。

**示例（默认 HashSet）：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
print(hashSetOps.create("usedIps"))
```

**示例（LinkedHashSet + 最大 500 条 + MMKV 持久化）：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
hashSetOps.create("usedIps", true, 500, "mmkv", "used_ips_data", "")
```

**示例（LinkedHashSet + 最大 1000 条 + SP 持久化）：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
hashSetOps.create("usedIps", true, 1000, "sp", "used_ips_data", "xfx_hashset")
```

#### remove()

删除 HashSet 实例（单参数重载，删除整个实例而非元素）。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 删除成功返回 true |

**示例：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
print(hashSetOps.remove("usedIps"))
```

### 2. 元素操作

#### add()

向集合中添加元素。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |
| element | String | 否 | 元素内容，空则存入空串 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 元素原本不存在并成功添加返回 true，已存在返回 false |

**示例：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
hashSetOps.create("usedIps")
print(hashSetOps.add("usedIps", "192.168.1.1"))
```

#### remove()

从集合中移除指定元素（双参数重载）。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |
| element | String | 否 | 要移除的元素 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 元素存在并成功移除返回 true |

**示例：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
hashSetOps.create("usedIps")
hashSetOps.add("usedIps", "192.168.1.1")
print(hashSetOps.remove("usedIps", "192.168.1.1"))
```

#### contains()

判断集合是否包含指定元素。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |
| element | String | 否 | 要判断的元素 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 包含返回 true |

**示例：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
hashSetOps.create("usedIps")
hashSetOps.add("usedIps", "192.168.1.1")
print(hashSetOps.contains("usedIps", "192.168.1.1"))
```

### 3. 状态查询

#### size()

获取集合大小。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |

| 返回值类型 | 说明 |
|-----------|------|
| Int | 大小，集合不存在返回 -1 |

**示例：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
hashSetOps.create("usedIps")
hashSetOps.add("usedIps", "192.168.1.1")
print(hashSetOps.size("usedIps"))
```

#### isEmpty()

判断集合是否为空。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 为空返回 true |

**示例：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
hashSetOps.create("usedIps")
print(hashSetOps.isEmpty("usedIps"))
```

#### getMaxSize()

获取集合配置的最大条数。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |

| 返回值类型 | 说明 |
|-----------|------|
| Int | 最大条数，0 表示不限制，集合不存在返回 -1 |

**示例：**
```lua

print(hashSetOps.getMaxSize("usedIps"))  -- 500
```

#### isOrdered()

判断集合是否为有序（LinkedHashSet）模式。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 有序返回 true，集合不存在返回 false |

**示例：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
hashSetOps.create("usedIps", true, 0, "none", "", "")
print(hashSetOps.isOrdered("usedIps"))  -- true
```

### 4. 持久化

#### save() / load()

使用 create 时配置的 persistType / persistKey 手动保存或加载。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 成功返回 true，未配置持久化或失败返回 false |

**示例：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
hashSetOps.create("usedIps", true, 0, "mmkv", "used_ips", "")
hashSetOps.add("usedIps", "192.168.1.1")
print(hashSetOps.save("usedIps"))
print(hashSetOps.load("usedIps"))
```

#### saveToMmkv() / loadFromMmkv()

使用 `MMKVOps` 将集合保存到 MMKV 或从中加载。

| 方法 | 参数 | 说明 |
|------|------|------|
| saveToMmkv(id, persistKey) | id: 集合 ID；persistKey: MMKV 键 | 保存整份 JSON 数组 |
| loadFromMmkv(id, persistKey) | 同上 | 加载并覆盖内存数据 |

**示例：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
hashSetOps.create("usedIps")
hashSetOps.add("usedIps", "192.168.1.1")
hashSetOps.saveToMmkv("usedIps", "manual_used_ips")

-- 重启脚本后恢复
hashSetOps.create("usedIps")
hashSetOps.loadFromMmkv("usedIps", "manual_used_ips")
print(hashSetOps.contains("usedIps", "192.168.1.1"))
```

#### saveToSp() / loadFromSp()

使用 `SPStaticOps` 将集合保存到 SharedPreferences 或从中加载。

| 方法 | 参数 | 说明 |
|------|------|------|
| saveToSp(id, persistKey, spName) | spName 可为空串（默认 SP） | 保存整份 JSON 数组 |
| loadFromSp(id, persistKey, spName) | 同上 | 加载并覆盖内存数据 |

**示例：**
```lua
local hashSetOps = UtilsMain.hashSetOps()
hashSetOps.create("usedIps")
hashSetOps.add("usedIps", "192.168.1.1")
hashSetOps.saveToSp("usedIps", "used_ips", "xfx_hashset")

-- 重启脚本后恢复
hashSetOps.create("usedIps")
hashSetOps.loadFromSp("usedIps", "used_ips", "xfx_hashset")
print(hashSetOps.contains("usedIps", "192.168.1.1"))
```

### 5. 清理与导出

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |

| 返回值类型 | 说明 |

**示例：**
```lua
local hashSetOps = UtilsMain.hashSetOps()

print(hashSetOps.clear("usedIps"))
```

#### addAll()

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 集合唯一标识 |

## 完整示例

```lua
import('com.nx.assist.lua.LuaEngine')
local loader = LuaEngine.loadApk('xfxPlugin-release.apk'); assert(loader)
local context = LuaEngine.getContext()
local UtilsMain = loader.loadClass('com.xfx.plugin.UtilsMain'); assert(UtilsMain)
UtilsMain.init(context)

local hashSetOps = UtilsMain.hashSetOps()

-- 创建
print("create:", hashSetOps.create("usedIps"))

-- 添加
print("add1:", hashSetOps.add("usedIps", "192.168.1.1"))
print("add2:", hashSetOps.add("usedIps", "10.0.0.1"))
print("add duplicate:", hashSetOps.add("usedIps", "192.168.1.1"))  -- false

-- 包含判断
print("contains:", hashSetOps.contains("usedIps", "10.0.0.1"))

-- 状态
print("size:", hashSetOps.size("usedIps"))
print("isEmpty:", hashSetOps.isEmpty("usedIps"))

-- 批量添加
print("addAll:", hashSetOps.addAll("usedIps", '["1.1.1.1","2.2.2.2"]'))

-- 导出 JSON
print("json:", hashSetOps.toJsonArray("usedIps"))

-- 移除元素
print("remove element:", hashSetOps.remove("usedIps", "10.0.0.1"))

-- 清空
print("clear:", hashSetOps.clear("usedIps"))

-- 删除实例
print("remove instance:", hashSetOps.remove("usedIps"))
```

## OOP 示例（IP 去重 + 持久化）

```lua
local xfxModule = require('lib/XfxPlugin')
local XFX = xfxModule:new({ apkName = 'xfxPlugin-release.apk' })

local hashSetOps = XFX:getOps('hashSetOps')

local SET_ID = 'usedIps'
-- 有序 + 最多 500 条 + MMKV 持久化，脚本重启后历史 IP 自动恢复
hashSetOps.create(SET_ID, true, 500, "mmkv", "used_ips_data", "")

local function getUniqueIp(fetchIp)
    local ip = fetchIp()
    local retry = 0
    while hashSetOps.contains(SET_ID, ip) do
        retry = retry + 1
        if retry > 10 then
            error('换 IP 超过最大重试次数')
        end
        ip = fetchIp()
    end
    hashSetOps.add(SET_ID, ip)  -- 自动持久化到 MMKV
    return ip
end

local ip = getUniqueIp(function() return '192.168.' .. math.random(1, 255) .. '.1' end)
print('本次使用 IP：' .. ip)
print('历史 IP 数量：' .. hashSetOps.size(SET_ID))
print('历史 IP 列表：' .. hashSetOps.toJsonArray(SET_ID))
```

## LinkedHashSet 有序 + 最大条数限制

```lua
import('com.nx.assist.lua.LuaEngine')
local loader = LuaEngine.loadApk('xfxPlugin-release.apk'); assert(loader)
local UtilsMain = loader.loadClass('com.xfx.plugin.UtilsMain'); assert(UtilsMain)
UtilsMain.init(LuaEngine.getContext())

local hashSetOps = UtilsMain.hashSetOps()
local SET_ID = 'usedIps'

-- 有序模式，最多保留 3 条 IP，超出自动删除最早插入的
hashSetOps.create(SET_ID, true, 3, "none", "", "")

hashSetOps.add(SET_ID, '192.168.1.1')
hashSetOps.add(SET_ID, '192.168.1.2')
hashSetOps.add(SET_ID, '192.168.1.3')
print('size:', hashSetOps.size(SET_ID))              -- 3
print('json:', hashSetOps.toJsonArray(SET_ID))      -- ["192.168.1.1","192.168.1.2","192.168.1.3"]

-- 第 4 条插入后，最早的 192.168.1.1 被自动删除
hashSetOps.add(SET_ID, '10.0.0.1')
print('size:', hashSetOps.size(SET_ID))              -- 3
print('contains 192.168.1.1:', hashSetOps.contains(SET_ID, '192.168.1.1'))  -- false
print('json:', hashSetOps.toJsonArray(SET_ID))      -- ["192.168.1.2","192.168.1.3","10.0.0.1"]
```

## 手动使用 mmkvOps / SPStaticOps 持久化

```lua
import('com.nx.assist.lua.LuaEngine')
local loader = LuaEngine.loadApk('xfxPlugin-release.apk'); assert(loader)
local UtilsMain = loader.loadClass('com.xfx.plugin.UtilsMain'); assert(UtilsMain)
UtilsMain.init(LuaEngine.getContext())

local hashSetOps = UtilsMain.hashSetOps()
local mmkvOps = UtilsMain.mmkvOps()
local spStaticOps = UtilsMain.spStaticOps()

local SET_ID = 'usedIps'
hashSetOps.create(SET_ID)

hashSetOps.add(SET_ID, '192.168.1.1')
hashSetOps.add(SET_ID, '10.0.0.1')

-- 方式一：HashSetOps 封装方法
hashSetOps.saveToMmkv(SET_ID, 'manual_used_ips')

-- 方式二：直接使用 mmkvOps
mmkvOps.put('manual_used_ips', hashSetOps.toJsonArray(SET_ID))
local json = mmkvOps.get('manual_used_ips', '[]')
hashSetOps.clear(SET_ID)
hashSetOps.addAll(SET_ID, json)

-- 方式三：直接使用 SPStaticOps
spStaticOps.setInstance('xfx_hashset')
spStaticOps.put('manual_used_ips', hashSetOps.toJsonArray(SET_ID))
```

## HashSet 与 LinkedHashSet 对比

| 对比维度 | HashSet（默认 create） | LinkedHashSet（ordered=true） |
| ---- | ---- | ---- |
| **底层实现** | ConcurrentHashMap.newKeySet() | LinkedHashSet + synchronized |
| **遍历顺序** | 不保证 | 按插入顺序 |
| **最大条数** | 不支持 | 支持 maxSize，超出删最早 |
| **并发性能** | 更高 | 略低（单锁同步） |
| **典型用途** | 通用去重集合 | IP 历史记录、容量受限去重 |

## 注意事项

- **ID 一致性**：所有操作需使用同一集合 ID；未创建的 ID 将导致操作失败。
- **元素类型**：仅支持 String 类型元素；其他类型需先转为字符串。
- **remove 重载**：`remove(id)` 删除整个实例；`remove(id, element)` 移除单个元素。
- **add 返回值**：返回 true 表示元素是新加入的，false 表示元素已存在。
- **maxSize 限制**：仅有序模式生效；超出后按**插入顺序**删除最早的数据。
- **持久化格式**：整份集合以 JSON 数组字符串存储在单个 persistKey 下；启用自动持久化后，每次 add/remove/clear 都会写入本地。
- **create 幂等**：同一 ID 重复 create 不会更新配置，需先 `remove(id)` 再重新 create。
- **并发安全**：HashSet 模式使用 ConcurrentHashMap.newKeySet()；LinkedHashSet 模式使用 synchronized，多线程场景建议业务层加锁。
- **生命周期**：脚本长期运行时建议在不使用时调用 `remove(id)` 释放实例，避免内存占用。
