## 模块简介

`PaddleOcrOps` 提供 PaddleOCR 文字识别接口，适合安卓移动端截图、局部图片、游戏 UI 文本识别等场景。

> 已接入 [NCNN](https://github.com/Tencent/ncnn/) 推理引擎，OCR 默认使用 CPU 推理，优先保证识别稳定性。


<!-- > 实现以 [nihui/ncnn-android-ppocrv5](https://github.com/nihui/ncnn-android-ppocrv5) 为主要参考，使用 NCNN Android Vulkan 推理，插件侧封装为基础类型参数和 JSON 字符串返回。 -->

## 模型要求

插件已内置 PP-OCR **v4 / v5 / v6** 三套 `PP-OCR mobile NCNN` 模型。

调用 `init()` 默认加载 **v6**；
也可通过 `init(version)` 指定版本、`initFromDir(modelDir)` 按目录自动发现模型，或通过带 5 个文件路径的 `init(...)` 加载自定义模型。

| 版本 | assets 目录 | 释放目录名 | 说明 |
| --- | --- | --- | --- |
| `v4` | `ppocrv4` | `xfx_ppocrv4` | 旧版 CRNN 识别链路，32 高输入 |
| `v5` | `ppocrv5` | `xfx_ppocrv5` | 官方 NCNN mobile 模型 |
| `v6` | `ppocrv6` | `xfx_ppocrv6` | 默认版本，识别精度更高 |

`version` 参数忽略大小写，支持 `v4`、`V5`、`6` 等写法；传入不支持的版本会返回 `1001` 参数错误。

自定义模型需要提供以下文件：

| 文件 | 说明 |
| --- | --- |
| `det.ncnn.param` | 文本检测模型结构 |
| `det.ncnn.bin` | 文本检测模型权重 |
| `rec.ncnn.param` | 文字识别模型结构 |
| `rec.ncnn.bin` | 文字识别模型权重 |
| `keys.txt` | 文字字典文件，每行一个字符 |

推荐优先使用 mobile 模型(server 模型体积更大，初始化和识别耗时更高)。

自定义模型需要与当前 PaddleOCR/NCNN 预处理和 CTC 解码链路匹配。

PP-OCRv4 这类旧 CRNN 识别模型会自动使用 32 高识别输入和 FP32 识别分支；

如果仍返回文字乱码，请优先确认 `rec.param`、`rec.bin` 与 `keys.txt` 同源，并确认模型不依赖宿主 APK 私有的额外预处理或后处理。

## 功能概览

- **多版本内置模型**：内置 v4 / v5 / v6，默认 v6，可通过 `init(version)` 切换
- **目录自动发现**：`initFromDir(modelDir)` 自动匹配 det/rec/keys 共 5 个文件
- **自定义模型**：加载外部检测模型、识别模型和字典
- **OCR 识别**：对 Bitmap 执行文字检测与识别，返回 JSON 字符串
- **阈值控制**：支持检测阈值、文本框扩展比例、识别分数阈值、识别输入最大宽度
- **性能优化**：识别阶段并行执行，默认限制识别输入最大宽度为 320
- **状态查询**：初始化状态、native 版本、最近错误码和错误消息
- **资源管理**：释放 native 模型资源，适配脚本结束、Activity 销毁、切换模型等场景

## 初始化与模型

### init()

使用内置默认模型初始化（当前默认 **v6**）。

首次调用会把插件 APK 内的模型复制到可写目录，再由 NCNN 加载。

```lua
local PaddleOps = UtilsMain.paddleOcrOps()

local ok = PaddleOps.init()
if not ok then
  print(PaddleOps.getLastErrorCode())
  print(PaddleOps.getLastErrorMessage())
end
```

### init(version)

使用指定版本的内置模型初始化（`v4` / `v5` / `v6`）

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| version | String | 是 | 模型版本（忽略大小写；为空时用默认 v6） |


```lua
local PaddleOps = UtilsMain.paddleOcrOps()

-- 按版本初始化内置模型
local okV6 = PaddleOps.init("v6")   -- 与 init() 等价
local okV5 = PaddleOps.init("V5")   -- 使用 v5 模型
local okV4 = PaddleOps.init("4")    -- 使用 v4 模型
print(PaddleOps.getLoadedVersion()) -- 输出: v4 / v5 / v6

```


### init(...) 自定义模型（显式路径）

加载自定义 NCNN 模型，手动指定全部文件路径。

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| detParamPath | String | 是 | 文本检测模型 param 文件路径 |
| recParamPath | String | 是 | 文字识别模型 param 文件路径 |
| detBinPath | String | 是 | 文本检测模型 bin 文件路径 |
| recBinPath | String | 是 | 文字识别模型 bin 文件路径 |
| keyTxtPath | String | 是 | 字典文件路径，每行一个字符 |

```lua
local PaddleOps = UtilsMain.paddleOcrOps()

-- 等价于 PaddleOps.init("/sdcard/paddle")，只是手动指定文件
local ok = PaddleOps.init(
  "/sdcard/paddle/PP_OCRv5_mobile_det.ncnn.param",
  "/sdcard/paddle/PP_OCRv5_mobile_rec.ncnn.param",
  "/sdcard/paddle/PP_OCRv5_mobile_det.ncnn.bin",
  "/sdcard/paddle/PP_OCRv5_mobile_rec.ncnn.bin",
  "/sdcard/paddle/keys.txt"
)

print(ok)
print(PaddleOps.getLastErrorCode())
print(PaddleOps.getLastErrorMessage())
-- print(PaddleOps.getDefaultModelDir())      -- /data/user/0/com.nx.nxprojit/files/assets/xfx_ppocrv6
```


### initFromDir(modelDirPath)

自动在指定目录中寻找 NCNN 所需的 5 个文件并加载。



| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| modelDirPath | String | 是 | 包含 det/rec/keys 共 5 个文件的目录路径 |

```lua
local PaddleOps = UtilsMain.paddleOcrOps()

-- 按目录自动发现模型文件（推荐自定义模型写法）
local okDir = PaddleOps.initFromDir("/sdcard/paddle")
print(okDir)
print(PaddleOps.getLastErrorCode())
print(PaddleOps.getLastErrorMessage())
```

目录自动发现规则（仅扫描该目录下文件，不递归子目录）：

| 角色 | 匹配方式 |
| --- | --- |
| det param / bin | 文件名分段含 `det`，扩展名为 `.param` / `.bin`（忽略 `cls`） |
| rec param / bin | 文件名分段含 `rec`，扩展名为 `.param` / `.bin`（忽略 `cls`） |
| keys | 优先 `keys.txt`，否则文件名含 `keys` 的 `.txt` |

支持常见命名，例如：

- `PP_OCRv5_mobile_det.ncnn.param` + `PP_OCRv5_mobile_det.ncnn.bin`
- `det.param` + `det.bin` + `rec.param` + `rec.bin` + `keys.txt`

## OCR 识别

### detect()

使用默认阈值执行 OCR 识别。

> 范围识别比全屏识别要更准确，速度也更快。大多数场景下，建议采用范围识别。

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| bitmap | Bitmap | 是 | 待识别 Bitmap，不能为 nil 或已回收 |

```lua
local bmp = LuaEngine.snapShot(0, 0, 280, 271)
local resultJson = PaddleOps.detect(bmp)
LuaEngine.releaseBmp(bmp)
print(resultJson)
```

### detect() 

使用自定义阈值执行 OCR 识别。


| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| bitmap | Bitmap | 是 | 待识别 Bitmap |
| boxThreshold | Float | 是 | 文本检测阈值，范围 `0.0` 到 `1.0`，默认 `0.30` |
| unclipRatio | Float | 是 | 文本框扩展比例，必须大于 `0`，默认 `1.60` |
| recScoreThreshold | Float | 是 | 识别分数阈值，范围 `0.0` 到 `1.0`，默认 `0.0`；用于过滤`score`低分结果 |
| maxRecWidth | Int | 否 | 识别模型输入最大宽度，默认 `320`，范围 `8` 到 `2048`；越小识别速度越快，长文本精度可能下降 |

> maxRecWidth 默认 320，适合移动端速度优先；长文本识别不完整时可尝试 480 或 640。

```lua
local resultJson = PaddleOps.detect(bmp, 0.30, 1.60, 0.6)
local hpJson = PaddleOps.detect(bmp, 0.5, 1.0, 0.8, 80)   -- 竖屏：快速识别
local spJson = PaddleOps.detect(bmp, 0.3, 1.0, 0.5, 120)   -- 横屏：快速识别
```

## 返回说明

成功时返回 JSON 字符串：

```json
{
  "code": 0,
  "message": "ok",
  "imageWidth": 280,
  "imageHeight": 271,
  "count": 2,
  "elapsedMs": 86,
  "data": [
    {
      "label": "开始任务",
      "score": 0.91,
      "boxScore": 0.83,
      "left": 32,
      "top": 18,
      "right": 128,
      "bottom": 48,
      "width": 96,
      "height": 30,
      "centerX": 80,
      "centerY": 33
    }
  ]
}
```

失败时返回 JSON 字符串：

```json
{
  "code": 3001,
  "message": "PaddleOCR is not initialized",
  "count": 0,
  "data": []
}
```

## 错误码

| code | 说明 | 常见原因 |
| --- | --- | --- |
| `0` | 成功 | 正常 |
| `1001` | 参数错误 | 空路径、Bitmap 无效、阈值超出范围、不支持的模型版本 |
| `1002` | 文件不存在 | param、bin 或 keys 路径错误，或默认模型释放失败 |
| `2001` | native 库加载失败 | APK 未包含 `libxfx_yolo.so` 或 ABI 不匹配 |
| `2002` | native 初始化失败 | 模型格式不匹配、字典为空、native 返回空句柄 |
| `3001` | 未初始化 | 调用 detect 前没有成功 init |
| `3002` | 识别失败 | Bitmap 格式不支持、native 推理异常 |

## 资源管理

### release()

释放当前 native 模型资源。

切换模型、脚本结束或 Activity 销毁前建议调用。

**示例：**
```lua
local ok = PaddleOps.release()
printEx(ok)
```

## 状态与信息

### isInitialized()

查询是否已完成模型初始化。

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 已初始化返回 true |

**示例：**
```lua
print("已初始化:", PaddleOps.isInitialized())
```

### getLastErrorCode()

获取最近一次错误码。

| 返回值类型 | 说明 |
|-----------|------|
| Int | 错误码，参见错误码表 |

**示例：**
```lua
print("错误码:", PaddleOps.getLastErrorCode())
```

### getLastErrorMessage()

获取最近一次错误消息。

| 返回值类型 | 说明 |
|-----------|------|
| String | 错误消息文本 |

**示例：**
```lua
print("错误消息:", PaddleOps.getLastErrorMessage())
```

### status()

获取当前 OCR 模块状态（JSON 字符串）。

| 返回值类型 | 说明 |
|-----------|------|
| String | 当前状态 JSON，含 `modelVersion` 字段 |

**示例：**
```lua
print("状态:", PaddleOps.status())
-- 输出: {"code":0,"message":"ok","initialized":true,"nativeLoaded":true,"modelVersion":"v6"}
```

### getLoadedVersion()

获取当前已加载的内置模型版本。

通过 `init(version)` 或 `init()` 成功初始化后才有值；自定义模型路径初始化时不设置版本，返回空字符串。

| 返回值类型 | 说明 |
|-----------|------|
| String | 已加载版本，如 `v4` / `v5` / `v6`；未加载返回空字符串 |

**示例：**
```lua
PaddleOps.init("v5")
print(PaddleOps.getLoadedVersion()) -- 输出: v5
```

### nativeVersion()

获取 native 库版本号。

| 返回值类型 | 说明 |
|-----------|------|
| String | native 库版本，失败返回空字符串 |

**示例：**
```lua
print("native库版本:", PaddleOps.nativeVersion())
```

```lua
    top = math.floor(screenHeight / 4),

    height = math.floor(screenHeight / 6),
    isShowProgressBar = true,
    isBottomInfo = true,
    isControlBtn = true,
})

XFX:setFloatLogTitle('XFX PaddleOCR Demo')
XFX:setFloatLogInfo('准备初始化 OCR...')

-- 获取 PaddleOps 对象
local PaddleOps = XFX:getOps('paddleOcrOps'); assert(PaddleOps)

-- 初始化（使用内置默认模型）
local ok = PaddleOps.init()

print(ok)
print(PaddleOps.getLastErrorCode())
print(PaddleOps.getLastErrorMessage())
-- print(PaddleOps.getDefaultModelDir())

if not ok then
  XFX:loge('OCR 初始化失败: ' .. tostring(PaddleOps.getLastErrorMessage()))
  XFX:setFloatLogInfo('OCR 初始化失败')
else
  XFX:logs('OCR 初始化成功')
  XFX:setFloatLogInfo('OCR 初始化成功，开始识别...')

  -- 截图识别
  local bmp = LuaEngine.snapShot(0, 0, 0, 0)
  local resultJson = PaddleOps.detect(bmp, 0.30, 1.0, 0.5, 120)
  bmp.recycle()
  XFX:logi('识别结果: ' .. tostring(resultJson))
end

-- 释放资源
PaddleOps.release()
XFX:logs('OCR 资源已释放')

sleep(5000)

XFX:closeFloatLog()
```


## 注意事项

- 初始化时，内置模型会释放到应用私有目录；失败后会尝试释放到备用可写目录。
- 字符索引字典必须与识别模型匹配；
- 固定使用 CPU 推理，避免部分设备 Vulkan/GPU 路径导致识别内容异常。
- 切换模型、脚本结束或 Activity 销毁前建议调用 `release()`。
