所有工具类均通过 `UtilsMain.xxxOps()` 获取。

支持Auto.js及变种(AutoX.js)、Aiwork、AirsSript、Cheese、Easyclick、懒人精灵、按键精灵、触动精灵、自动化编辑器等安卓脚本框架调用。

## 懒人精灵高级版(安卓)

### 直接调用

```lua
import('com.nx.assist.lua.LuaEngine')

-- 加载 APK 文件
local loader = LuaEngine.loadApk('xfxPlugin-release.apk')
if not loader then
    error('Failed to load APK')
end

-- 加载插件主类
local UtilsMain = loader.loadClass('com.xfx.plugin.UtilsMain')
if not UtilsMain then
    error('Failed to load Class')
end

-- 设置插件的全局上下文（可选）
UtilsMain.init(LuaEngine.getContext())


-- 获取字符串工具
local strOps = UtilsMain.stringOps()
print(strOps.upperFirstLetter('hello'))  -- 输出: Hello
print(strOps.isEmpty('test'))             -- 输出: false
print(strOps.reverse('abc123'))           -- 输出: 321cba

-- 获取文件工具
local fileOps = UtilsMain.fileOps()
local basePath = '/sdcard/xfx_demo/'
local filePath = basePath .. 'hello.txt'
fileOps.create(basePath)
fileOps.write(filePath, 'Hello Lua!')
print('内容: ' .. fileOps.read(filePath))

-- 获取时间工具
local timeOps = UtilsMain.timeOps()
print('当前时间: ' .. timeOps.getNowString()) -- 输出: 当前时间: 2025-12-12 16:57:13
```

### 通过`XfxPlugin.lua`插件调用(OOP方式)

由于悬浮窗日志、MQTT等类模块初始化的时候需要的参数比较多，封装了`XfxPlugin.lua`插件方便调用。

调用此插件后，`Ops` 对象统一提供 `call(methodName, vararg args)` 方法来执行对应的静态方法。

```lua
import('com.nx.assist.lua.LuaEngine')

-- 加载 XfxPlugin.lua 插件
local xfxModule = require('lib/XfxPlugin')

-- 创建 XFX 对象实例
local XFX = xfxModule:new({
    apkPath = 'xfxPlugin-release.apk',  -- APK 文件名/完整路径
    context = LuaEngine.getContext(),  -- 上下文对象（不支持最高权限）
})

-- 方法 1：通过 call 方法调用 DeviceOps 的方法
print('\n--- 通过 call 方法调用 ---')
local ok1, isRooted = pcall(function()
    return XFX:call('deviceOps', 'isDeviceRooted')
end)
local ok2, manufacturer = pcall(function()
    return XFX:call('deviceOps', 'getManufacturer')
end)

if ok1 and ok2 then
    print('设备是否 Root: ' .. tostring(isRooted))
    print('设备制造商: ' .. tostring(manufacturer))
else
    print('获取设备信息失败')
end

-- 可携带参数：
print('当前时间： '.. XFX:call('timeOps', 'getNowString','HH:mm:ss'))

-- 方法 2：直接获取 deviceOps 对象，然后调用方法（更直观，推荐）
print('\n--- 直接获取 deviceOps 对象 ---')
local deviceOps = XFX:getOps('deviceOps')
if not deviceOps then
    error('无法获取 deviceOps 对象')
end

-- 判断 Root / ADB
print('\n【权限和调试信息】')
local isRooted = deviceOps.isDeviceRooted()
local isAdbEnabled = deviceOps.isAdbEnabled()
local isDevSettingsEnabled = deviceOps.isDevelopmentSettingsEnabled()
print('是否 Root: ' .. tostring(isRooted))
print('ADB 是否开启: ' .. tostring(isAdbEnabled))
print('开发者选项是否开启: ' .. tostring(isDevSettingsEnabled))
```

## Auto.js

```js
/**
 * xfxPlugin - Auto.js Pro 调用示例
 * 
 * 使用前请确保插件文件已放置：
 * 
 * 方式1：从脚本所在目录加载（推荐）
 * - 将 xfxPlugin-release.dex 文件放到脚本所在目录的 xfxPlugin/ 下
 * - 将 lib 目录放到脚本所在目录的 xfxPlugin/lib/ 下 （包含各架构的 so 库）
 * 方式2：将插件和so文件放在自定义路径
 */

// 引入插件模块
var XFX = require('./xfxPlugin/xfxPlugin.js');

// ====================== 加载并初始化插件 ======================
try {
    // 加载插件，返回 UtilsMain 代理对象
    var UtilsMain = XFX.load(context, files.cwd() + "/xfxPlugin");
    
    // 初始化插件（现在可以直接调用静态方法了！）
    UtilsMain.init(context);
    
    toastLog("插件初始化成功");
    log("插件版本：" + UtilsMain.version());
} catch (e) {
    toastLog("初始化失败：" + e.message);
    log("错误详情：", e.stack);
    exit();
}


// ====================== 使用示例 ======================
// 现在可以直接调用 UtilsMain 的静态方法，就像调用 Java 类一样！

// 1. 字符串工具
try {
    var strOps = UtilsMain.stringOps();
    log("首字母大写：" + strOps.upperFirstLetter("hello"));

    log("反转字符串：" + strOps.reverse("abc123"));
```

## EasyClick（安卓）

[官方文档](https://ieasyclick.com/docs/zh-cn/funcs/global/)

> `UtilsMain` 是 Kotlin `object` 单例，**不能** `new`，应通过 `importClass` 后直接调静态方法（与 AiWork 一致）。

```js
function main() {
    // 将插件 apk 放到项目的 plugins 目录；loadDex 必须写在 main 方法内
    let s = loadDex("xfxPlugin-release.apk");
    // 或者从自定义目录加载：
    // let s = loadDex("/sdcard/xfxPlugin-release.apk");

    if (!s) {
        logd("小飞侠插件加载失败");
        return;
    }
    logd("小飞侠插件加载成功");

    // 导入入口类，直接调用静态方法（不要 new）
    importClass(com.xfx.plugin.UtilsMain);
    UtilsMain.init(context);

    // 获取字符串工具
    let strOps = UtilsMain.stringOps();
    logd(strOps.upperFirstLetter('hello'));  // 输出: Hello
    logd(strOps.isEmpty('test'));             // 输出: false
    logd(strOps.reverse('abc123'));           // 输出: 321cba
}

main();
```

## ⚠️ 重要注意事项

**参数必须完整**：

- 调用插件接口时，所有方法必须提供完整的参数，参数数量和类型必须与插件的接口完全一致（必须显式传递所有参数）
- 如果参数为可选（可为 null），必须传递 `""` 而不是省略
- 示例：`addButton` 需要10个参数，即使后面的参数都有默认值，也必须全部传递
