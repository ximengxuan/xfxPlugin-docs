## 模块简介

`MySQLOps` 提供 MySQL(v8.0.33) 数据库操作，包含连接管理、CRUD、事务与简单异步写入。使用连接池管理连接，支持多数据库连接（通过连接名区分）。

> ⚠️ 绝大多数场景下，移动端脚本不建议直接连接 MySQL；仅特殊场景可考虑，且需谨慎。

## 功能概览

- **连接管理**：connect 按连接名管理多个数据库连接，支持连接池配置；close 断开连接
- **错误处理**：getLastError 获取最后一次操作错误信息
- **CRUD**：execSql、insert、update、delete、query（结果 JSON）
- **事务**：手动 begin/setSuccessful/end 或一次性 runInTransaction
- **异步**：execSqlAsync 简单线程 fire-and-forget
- **参数规约**：所有方法必须按声明顺序显式传递全部参数；列表一律用 JSON 字符串

## 参数/数据格式约定

- `sqlArrayJson`、`argsJson`、`whereArgsJson`：形如 `["value1","value2"]` 的 JSON 数组字符串
- `valuesJson`：形如 `{"col":"value","num":1,"flag":true}` 的 JSON 对象字符串
- `query` 返回值：JSON 数组字符串，每行是对象，例如 `[{"id":1,"title":"a"}]`

## API 接口列表

### connect()

连接到 MySQL 数据库。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| connName | String | 是 | 连接名称（用于区分多个数据库连接） |
| host | String | 是 | 数据库主机地址 |
| port | Int | 是 | 数据库端口（默认 3306） |
| database | String | 是 | 数据库名 |
| username | String | 是 | 用户名 |
| password | String | 是 | 密码 |
| charset | String? | 否 | 字符集（默认 utf8mb4） |
| connectTimeout | Int? | 否 | 连接超时（秒，默认 10） |
| socketTimeout | Int? | 否 | Socket 超时（秒，默认 30） |
| useSSL | Boolean? | 否 | 是否使用 SSL（默认 false） |
| allowPublicKeyRetrieval | Boolean? | 否 | 是否允许公钥检索（默认 true） |
| minPoolSize | Int? | 否 | 连接池最小连接数（默认 1） |
| maxPoolSize | Int? | 否 | 连接池最大连接数（默认 5） |

**返回：** Boolean

**示例：**

**方式一：简化调用（推荐，使用默认配置）**
```lua
local mysqlOps = UtilsMain.mysqlOps()

-- 连接到数据库（只传必需参数，使用默认配置）
-- 数据库不存在也会返回false，但不会抛出异常
print(mysqlOps.connect(
    "default",      -- 连接名
    "192.168.1.100", -- 主机
    3306,           -- 端口
    "testdb",       -- 数据库名
    "root",         -- 用户名
    "password"      -- 密码
))
```

**方式二：完整调用（自定义所有配置）**
```lua
local mysqlOps = UtilsMain.mysqlOps()

-- 连接到数据库（传递所有参数）
print(mysqlOps.connect(
    "default",      -- 连接名
    "192.168.1.100", -- 主机
    3306,           -- 端口
    "testdb",       -- 数据库名
    "root",         -- 用户名
    "password",     -- 密码
    "UTF-8",        -- 字符集（字符串，不能使用单引号）
    10,             -- 连接超时（秒）
    30,             -- Socket 超时（秒）
    false,          -- 是否使用 SSL（布尔值）
    true,           -- 是否允许公钥检索（布尔值）
    1,              -- 最小连接数
    5               -- 最大连接数
))
```

> ⚠️ **重要提示**：
> - 如果使用完整调用方式，必须传递所有 13 个参数，不能省略
> - 如果不需要自定义配置，建议使用简化调用方式（只传 6 个必需参数）

### getLastError()

获取最后一次操作的错误信息（用于调试连接失败等问题）。

| 返回值类型 | 说明 |
|-----------|------|
| String? | 错误信息字符串，如果没有错误则返回 null |

**示例：**
```lua
local result = mysqlOps.connect(connName, host, port, database, username, password)
if not result then
    local errorMsg = mysqlOps.getLastError()
    print("连接失败:", errorMsg)
end
```

### close()

断开数据库连接。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| connName | String | 是 | 连接名称 |

**返回：** Boolean

### execSql()

执行非查询 SQL。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| connName | String | 是 | 连接名称 |
| sql | String | 是 | SQL 语句 |
| argsJson | String? | 否 | 占位符参数 JSON 数组字符串 |

**返回：** Boolean

**示例：**
```lua
-- 执行更新语句
mysqlOps.execSql("default", "UPDATE users SET status = ? WHERE id = ?", jsonLib.encode({1, 100}))
```

### execSqlAsync()

在新线程执行非查询 SQL（fire-and-forget）。

参数同 `execSql`。  
**返回：** Boolean（仅表示线程已提交）

### query()

执行查询并返回 JSON 字符串。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| connName | String | 是 | 连接名称 |
| sql | String | 是 | 查询语句 |
| argsJson | String? | 否 | 占位符参数 JSON 数组字符串(SQL 中没有占位符 ? 的话可以不传) |

**返回：** JSON 字符串（数组）

```lua
-- SQL 中有占位符
local rowsJson = mysqlOps.query("default", "SELECT * FROM users WHERE status=?", jsonLib.encode({1}))
print(rowsJson)

-- 不传占位符参数
print(mysqlOps.query("default", "SELECT * FROM users"))
```

### insert()

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| connName | String | 是 | 连接名称 |
| table | String | 是 | 表名 |
| valuesJson | String | 是 | 列值 JSON 对象字符串 |

**返回：** Long（自增 ID，失败 -1）

```lua
local id = mysqlOps.insert("default", "users", jsonLib.encode({name = 'test', email = 'test@example.com', status = 1}))
print('插入的 ID：', id)
```

### update()

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| connName | String | 是 | 连接名称 |
| table | String | 是 | 表名 |
| valuesJson | String | 是 | 列值 JSON 对象字符串 |
| whereClause | String? | 否 | where 子句 |
| whereArgsJson | String? | 否 | where 占位符参数 JSON 数组 |

**返回：** Int（受影响行数，失败 -1）

**示例：**
```lua
local count = mysqlOps.update(
    "default", 
    "users", 
    jsonLib.encode({status = 0}), 
    "id = ?", 
    jsonLib.encode({100})
)
print('更新行数：', count)
```

### delete()

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| connName | String | 是 | 连接名称 |
| table | String | 是 | 表名 |
| whereClause | String? | 否 | where 子句 |
| whereArgsJson | String? | 否 | where 占位符参数 JSON 数组 |

**返回：** Int（受影响行数，失败 -1）

**示例：**
```lua
local count = mysqlOps.delete("default", "users", "status = ?", jsonLib.encode({0}))
print('删除行数：', count)
```

### beginTransaction() / setTransactionSuccessful() / endTransaction()

按顺序配合使用。

| 方法 | 返回 | 说明 |
|------|------|------|
| beginTransaction(connName) | Boolean | 开启事务 |
| setTransactionSuccessful(connName) | Boolean | 标记成功 |
| endTransaction(connName) | Boolean | 结束事务（提交或回滚） |

### runInTransaction()

一次性事务执行多条 SQL。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| connName | String | 是 | 连接名称 |
| sqlArrayJson | String | 是 | SQL 的 JSON 数组字符串 |

**返回：** Boolean

**示例：**
```lua
local txSqls = {
    "UPDATE users SET status=1 WHERE id=1",
    "UPDATE users SET status=1 WHERE id=2"
}
print('事务执行结果：', mysqlOps.runInTransaction("default", jsonLib.encode(txSqls)))
```

## Lua 调用示例

```lua
import('com.nx.assist.lua.LuaEngine')
local loader  = LuaEngine.loadApk('xfxPlugin-release.apk')
local context = LuaEngine.getContext()
local UtilsMain    = loader.loadClass('com.xfx.plugin.UtilsMain')

local mysqlOps = UtilsMain.mysqlOps()

local connName = "default"

-- 1) 连接到数据库
local result = mysqlOps.connect(
    connName,
    "192.168.1.100",  -- 主机地址
    3306,              -- 端口
    "testdb",          -- 数据库名
    "root",            -- 用户名
    "password"         -- 密码
)

if not result then
    -- 获取详细错误信息
    local errorMsg = mysqlOps.getLastError()
    error("连接失败，错误信息：", errorMsg)
else
    print("连接成功！")
end

-- 2) 检查表是否存在，不存在则创建
local tableName = "users"
if not mysqlOps.isExists(connName, tableName) then
    print("表不存在，开始创建表...")
    local createTableSql = string.format([[
        CREATE TABLE IF NOT EXISTS %s (
            id INT AUTO_INCREMENT PRIMARY KEY,
            name VARCHAR(100) NOT NULL,
            email VARCHAR(100) NOT NULL,
            status INT DEFAULT 0,
            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
            updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
            INDEX idx_status (status),
            INDEX idx_email (email)
        ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
    ]], tableName)
    
    local createResult = mysqlOps.execSql(connName, createTableSql, nil)
    if createResult then
        print("表创建成功！")
    else
        local errorMsg = mysqlOps.getLastError()
        print("表创建失败，错误信息：", errorMsg)
        mysqlOps.close(connName)
        return
    end
else
    print("表已存在")
end

-- 3) 插入数据
local id = mysqlOps.insert(connName, tableName, jsonLib.encode({
    name = 'test',
    email = 'test@example.com',
    status = 1
}))

if id > 0 then
    print('插入的 ID：', id)
else
    local errorMsg = mysqlOps.getLastError()
    print('插入失败，错误信息：', errorMsg)
end

-- 4) 查询（结果为 JSON 字符串）
local rowsJson = mysqlOps.query(connName, "SELECT * FROM " .. tableName .. " WHERE status=?", jsonLib.encode({1}))
print("查询结果：", rowsJson)

-- 5) 更新
sleep(1000)  -- 暂停N秒，再更新数据，不然表里的时间不会变动（可选）
if id > 0 then
    local count = mysqlOps.update(
        connName, 
        tableName, 
        jsonLib.encode({status = 0}), 
        "id = ?", 
        jsonLib.encode({id})
    )
    if count > 0 then
        print('更新行数：', count)
    else
        local errorMsg = mysqlOps.getLastError()
        print('更新失败，错误信息：', errorMsg)
    end
end

-- 6) 事务执行多条 SQL
local txSqls = {
    "UPDATE " .. tableName .. " SET status=1 WHERE id=1",
    "UPDATE " .. tableName .. " SET status=1 WHERE id=2"
}
local txResult = mysqlOps.runInTransaction(connName, jsonLib.encode(txSqls))
if txResult then
    print('事务执行成功')
else
    local errorMsg = mysqlOps.getLastError()
    print('事务执行失败，错误信息：', errorMsg)
end

-- 7) 查询（无占位符）
local allRows = mysqlOps.query(connName, "SELECT * FROM " .. tableName)
print("所有数据：", allRows)

-- 8) 断开连接
mysqlOps.close(connName)
print("已断开连接")
```

### 案例 2：批量更新配置（事务）

```lua
import('com.nx.assist.lua.LuaEngine')

local xfxModule = require('lib/XfxPlugin')
local XFX = xfxModule:new({
    apkName = 'xfxPlugin-release.apk',
    -- context = LuaEngine.getContext(),

local mysqlOps = XFX:getOps('mysqlOps')
local connName = 'configdb'

local result = mysqlOps.connect(
    connName,
    "192.168.1.100",

    "configdb",
    "root",
    "password"

if not result then
    local errorMsg = mysqlOps.getLastError()

if not mysqlOps.isExists(connName, tableName) then

    local createTableSql = string.format([[
        CREATE TABLE IF NOT EXISTS %s (

            key_name VARCHAR(100) NOT NULL UNIQUE,

        ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
    ]], tableName)
    
    local createResult = mysqlOps.execSql(connName, createTableSql, nil)
    if createResult then

            "INSERT INTO " .. tableName .. "(key_name, value, description) VALUES('env', 'dev', '环境配置')",
            "INSERT INTO " .. tableName .. "(key_name, value, description) VALUES('feature_x', 'off', '功能开关')",
            "INSERT INTO " .. tableName .. "(key_name, value, description) VALUES('timeout', '10', '超时时间')"
        }
        mysqlOps.runInTransaction(connName, jsonLib.encode(initSqls))
    else
        local errorMsg = mysqlOps.getLastError()
        print("表创建失败，错误信息：", errorMsg)
        mysqlOps.close(connName)
        return
    end
else
    print("表已存在")
end

-- 批量更新使用事务
local txSqls = {
    "UPDATE " .. tableName .. " SET value='prod' WHERE key_name='env'",
    "UPDATE " .. tableName .. " SET value='on' WHERE key_name='feature_x'",
    "UPDATE " .. tableName .. " SET value='30' WHERE key_name='timeout'"
}
local txResult = mysqlOps.runInTransaction(connName, jsonLib.encode(txSqls))
if txResult then
    print('事务执行成功')
else
    local errorMsg = mysqlOps.getLastError()
    print('事务执行失败，错误信息：', errorMsg)
end

-- 验证
print("更新后的配置：")
print(mysqlOps.query(connName, "SELECT * FROM " .. tableName))
-- [{"id":1,"key_name":"env","value":"prod","description":"环境配置","updated_at":"2025-12-12 10:22:47.0"},{"id":2,"key_name":"feature_x","value":"on","description":"功能开关","updated_at":"2025-12-12 10:22:47.0"},{"id":3,"key_name":"timeout","value":"30","description":"超时时间","updated_at":"2025-12-12 10:22:47.0"}]
```

```lua
print("已断开连接")
```

### 案例 4：多数据库连接管理

```lua
import('com.nx.assist.lua.LuaEngine')
```

## 注意事项

- **网络要求**：需要确保设备可以访问 MySQL 服务器（网络连接、防火墙配置等）。
- **连接管理**：使用连接名区分多个数据库连接，每个连接名对应一个独立的连接池。
- **连接池配置**：默认最小连接数 1，最大连接数 5，可根据实际需求调整。
- **事务管理**：使用事务时，确保调用 `beginTransaction`、`setTransactionSuccessful`（可选）、`endTransaction` 配对使用，或使用 `runInTransaction` 一次性执行。
- **参数传递**：所有方法必须按声明顺序显式传递全部参数，即使参数为可选（传 nil）。
- **错误处理**：方法返回 false、-1 或空数组表示操作失败，建议检查返回值。
- **线程安全**：连接池是线程安全的，可以在多线程环境中使用。
- **资源释放**：使用完毕后建议调用 `close` 释放连接资源。

## 特别提示

安卓端直接连 MySQL 是 “反常规设计”。

MySQL 是服务端数据库，安卓作为客户端，直接直连数据库存在核心问题：

- 安全风险：数据库账号密码需硬编码在安卓端（或本地存储），极易被反编译破解，导致数据库泄露 / 被篡改；
- 网络不稳定：移动端网络（4G/5G/WiFi）波动大，长连接易断开，重连逻辑复杂；
- 性能瓶颈：安卓设备性能远低于服务器，并发能力弱，连接池的 “复用” 收益有限；
- 架构违规：标准架构是 “安卓 ↔ 后端接口（Java/PHP/Node/GO/Python 等） ↔ MySQL”，后端统一处理数据库连接、权限、业务逻辑，安卓仅调用接口，这是行业通用的安全 / 性能最优解。
- 替代方案：如果数据量小、无需共享，直接用安卓内置的 SQLite（轻量、无网络依赖），无需连接远程 MySQL；
