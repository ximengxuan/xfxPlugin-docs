## 模块简介

`ImageOps` 封装图片加载、格式转换、缩放/旋转/倾斜/裁剪、模糊处理、水印、圆形/圆角、压缩与保存，以及图片信息查询等功能。

## 功能概览

- **加载**：路径/File 转 Bitmap，判断是否为图片
- **格式转换**：Bitmap ↔ ByteArray（JPEG/PNG/WEBP/默认）、Bitmap ↔ Mat、Bitmap ↔ Base64
- **基本处理**：缩放（尺寸/比例自动判断）、旋转、倾斜、裁剪、灰度化
- **模糊处理**：快速模糊、RenderScript 模糊、堆栈模糊
- **水印**：文字水印、图片水印
- **圆形/圆角**：转圆形、统一圆角
- **保存**：指定格式保存（路径或相册）
- **压缩**：按比例、按质量、按采样率压缩
- **信息**：获取文件信息 JSON、获取 Bitmap 信息 JSON
- **相似度比较**：均值哈希（aHash）、差值哈希（dHash）、直方图对比
- **克隆**：深拷贝 Bitmap
- **资源管理**：回收 Bitmap 资源

### 3. 基本处理

#### scale()

按比例或绝对尺寸缩放图片（自动判断）。如果 scaleW 和 scaleH 都在 0-1 之间，则按比例缩放；否则按绝对像素值缩放。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bitmap | Bitmap? | 是 | 源图片 |
| scaleW | Float | 是 | 宽度缩放比例（0-1之间）或新宽度（像素，大于1） |
| scaleH | Float | 是 | 高度缩放比例（0-1之间）或新高度（像素，大于1） |

| 返回值类型 | 说明 |
|-----------|------|
| Bitmap? | 缩放后的 Bitmap 或 null |

**示例：**
```lua
local imgOps = UtilsMain.imageOps()
local bmp = imgOps.getBitmap("/sdcard/pic.jpg")
-- 按比例缩放（缩小到50%）
local scaled1 = imgOps.scale(bmp, 0.5, 0.5)
-- 按绝对尺寸缩放（缩放到800x600）
local scaled2 = imgOps.scale(bmp, 800.0, 600.0)
```

#### rotate()

以中心轴旋转图片。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bitmap | Bitmap? | 是 | 源图片 |
| degrees | Int | 是 | 角度（正数为顺时针，负数为逆时针） |

| 返回值类型 | 说明 |
|-----------|------|
| Bitmap? | 旋转后的 Bitmap 或 null |

**示例：**
```lua
local imgOps = UtilsMain.imageOps()
local bmp = imgOps.getBitmap("/sdcard/pic.jpg")
local rotated = imgOps.rotate(bmp, 90)
```

倾斜图片。

| bitmap | Bitmap? | 是 | 源图片 |

| 返回值类型 | 说明 |

**示例：**

#### clip()

裁剪图片。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bitmap | Bitmap? | 是 | 源图片 |
| x | Int | 是 | 起始 x 坐标（左上角） |
| y | Int | 是 | 起始 y 坐标（左上角） |
| width | Int | 是 | 裁剪宽度（像素） |
| height | Int | 是 | 裁剪高度（像素） |

| 返回值类型 | 说明 |
|-----------|------|
| Bitmap? | 裁剪后的 Bitmap 或 null |

**示例：**
```lua
local imgOps = UtilsMain.imageOps()
local bmp = imgOps.getBitmap("/sdcard/pic.jpg")
-- 从 (100, 100) 位置裁剪 400x400 的区域
local clipped = imgOps.clip(bmp, 100, 100, 400, 400)
```

#### crop()

按边界坐标裁剪图片（与 `clip()` 区分）。

`crop(bitmap, l, t, r, b)` 使用 left/top/right/bottom 四边界，其中：
- `l`、`t` 为左上角坐标（包含）
- `r`、`b` 为右下角边界（不包含）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bitmap | Bitmap? | 是 | 源图片 |
| l | Int | 是 | 左边界 x 坐标（包含） |
| t | Int | 是 | 上边界 y 坐标（包含） |
| r | Int | 是 | 右边界 x 坐标（不包含） |
| b | Int | 是 | 下边界 y 坐标（不包含） |

| 返回值类型 | 说明 |
|-----------|------|
| Bitmap? | 裁剪后的 Bitmap 或 null |

**示例：**
```lua
local imgOps = UtilsMain.imageOps()
local bmp = imgOps.getBitmap("/sdcard/pic.jpg")
-- 按边界裁剪 [100,100] 到 [500,500) 的区域
local cropped = imgOps.crop(bmp, 100, 100, 500, 500)
```

### 9. 图片信息

#### getInfo()

获取图片文件信息 JSON（width/height/format/size/path/rotateDegree）。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| path | String? | 是 | 图片路径 |

| 返回值类型 | 说明 |
|-----------|------|
| String? | JSON 字符串或 null |

**JSON 格式：**
```json
{
  "width": 1920,
  "height": 1080,
  "format": "JPEG",
  "size": 1024000,
  "path": "/path/to/image.jpg",
  "rotateDegree": 90   // EXIF旋转角度
}
```

**示例：**
```lua
local imgOps = UtilsMain.imageOps()
print(imgOps.getInfo("/sdcard/pic.jpg"))
```

#### getInfo()

获取 Bitmap 对象的信息 JSON（width/height/byteCount/config/isRecycled/hasAlpha）。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bitmap | Bitmap? | 是 | Bitmap 对象 |

| 返回值类型 | 说明 |
|-----------|------|
| String? | JSON 字符串或 null |

**JSON 格式：**
```json
{
  "width": 1920,
  "height": 1080,
  "byteCount": 8294400,
  "config": "ARGB_8888",
  "isRecycled": false,
  "hasAlpha": true
}
```

**字段说明：**
- `width`: 图片宽度（像素）
- `height`: 图片高度（像素）
- `byteCount`: 占用的内存字节数
- `config`: 颜色配置（如 ARGB_8888、RGB_565 等）
- `isRecycled`: 是否已回收
- `hasAlpha`: 是否包含透明通道

**示例：**
```lua
local imgOps = UtilsMain.imageOps()
local bmp = imgOps.getBitmap("/sdcard/pic.jpg")
print(imgOps.getInfo(bmp))
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bmp1 | Bitmap? | 是 | 第一张图片 |
| bmp2 | Bitmap? | 是 | 第二张图片 |
| method | String? | 是 | 比较方法，支持 `"avgHash"`/`"aHash"`（均值哈希）、`"diffHash"`/`"dHash"`（差值哈希）、`"histogram"`/`"hist"`（直方图对比），不区分大小写 |

| 返回值类型 | 说明 |
|-----------|------|
| Float | 相似度（0.0-1.0），1.0 表示完全相同。出错或参数无效时返回 -1.0 |

**方法说明：**

- **avgHash / aHash**（均值哈希）：将图片缩放到 8×8 并转为灰度，计算像素均值，每位与均值比较生成 64 位哈希指纹，最后通过汉明距离计算相似度。适合比较整体结构相似的图片。
- **diffHash / dHash**（差值哈希）：将图片缩放到 9×8 并转为灰度，逐行比较相邻像素差异生成 64 位哈希指纹，最后通过汉明距离计算相似度。对图片的渐变和亮度变化更敏感。
- **histogram / hist**（直方图对比）：将图片缩放到 128×128，统计 RGB 各通道 64 bins 的颜色直方图，通过归一化互相关计算颜色分布相似度。适合比较颜色分布相似的图片。

**示例：**
```lua
local imgOps = UtilsMain.imageOps()
local bmp1 = imgOps.getBitmap("/sdcard/pic1.jpg")
local bmp2 = imgOps.getBitmap("/sdcard/pic2.jpg")

-- 均值哈希比较
local sim1 = imgOps.compare(bmp1, bmp2, "avgHash")
print("均值哈希相似度: " .. sim1)

-- 差值哈希比较
local sim2 = imgOps.compare(bmp1, bmp2, "diffHash")
print("差值哈希相似度: " .. sim2)

-- 直方图比较
local sim3 = imgOps.compare(bmp1, bmp2, "histogram")
print("直方图相似度: " .. sim3)
```

### 11. 克隆

#### clone()

克隆（深拷贝）一个 Bitmap，生成一个独立的全新副本。

新 Bitmap 与原 Bitmap 具有相同的像素数据和配置（Config、尺寸等），但彼此独立，互不影响。原 Bitmap 回收后克隆的 Bitmap 仍然可用。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| bitmap | Bitmap? | 是 | 需要克隆的 Bitmap 对象 |

| 返回值类型 | 说明 |
|-----------|------|
| Bitmap? | 克隆后的新 Bitmap 对象，失败（如参数为 null、已回收、或内存不足）返回 null |

**示例：**
```lua
local imgOps = UtilsMain.imageOps()
local bmp = imgOps.getBitmap("/sdcard/pic.jpg")

-- 克隆一份独立的副本
local clonedBmp = imgOps.clone(bmp)

-- 原图回收后，克隆的图片仍然可用
imgOps.recycle(bmp)
print(imgOps.getInfo(clonedBmp))  -- 仍然可以正常获取信息

imgOps.recycle(clonedBmp)
```

## 注意事项

- **内存**：Bitmap 处理可能占用大量内存，处理完成后建议使用 `recycle()` 回收资源以免出现内存泄露。
- **路径与权限**：读取/保存文件需存储权限；路径需可访问。 
- **格式支持**：支持的图片格式包括 JPEG、PNG、WEBP。
- **缩放参数**：`scale()` 方法会自动判断参数是比例（0-1）还是绝对尺寸（>1），使用时要确保参数类型正确。
- **Base64 转换**：`bitmapToBase64()` 返回纯 Base64 字符串（不含 data URI 前缀），`base64ToBitmap()` 会自动处理可能存在的 data URI 前缀。
