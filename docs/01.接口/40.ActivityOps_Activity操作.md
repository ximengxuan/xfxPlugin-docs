## 模块简介

`ActivityOps` 封装了常用的 Activity 相关操作，包括存在性判断、启动、返回桌面、存活判断与关闭。

> ⚠️ 不支持懒人精灵`最高权限`下运行(最高权限下获取不到Context)。

## 功能概览

- **上下文设置**：设置 Context（setContext，与 UtilsMain.init 二选一）
- **存在性判断**：判断指定 Activity 是否存在（isActivityExists）
- **启动与回桌面**：启动 Activity（startActivity）、回到桌面（homeActivity）
- **存活/结束**：判断 Activity 是否存活（isActivityAlive）、结束 Activity（finishActivity）

## API 接口列表

#### setContext()

按需设置 Context（建议传入 applicationContext），在未调用 `UtilsMain.init` 时也可提前注入。

> setContext()和 UtilsMain.init() 二选一调用即可。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| context | Context | 是 | 应用或 Activity 的 context，建议 applicationContext |

| 返回值类型 | 说明 |
|-----------|------|
| void | 无 |

**示例：**
```lua
import('com.nx.assist.lua.LuaEngine')

local actOps = UtilsMain.activityOps()

local context = LuaEngine.getContext()
actOps.setContext(context)  -- 传入已有的 context
```


### 1. Activity 判断与启动

#### isActivityExists(packageName: String, className: String)

判断 Activity 是否存在。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| packageName | String | 是 | 包名 |
| className | String | 是 | 完整类名 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 存在返回 true |

**示例：**
```lua
local actOps = UtilsMain.activityOps()
print(actOps.isActivityExists("com.example.app", "com.example.app.MainActivity"))
```

#### startActivity(intent: Intent)

启动指定 Intent 的 Activity。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| intent | Intent | 是 | 已构建好的 Intent |

| 返回值类型 | 说明 |
|-----------|------|
| void | 无返回值 |

**示例：**
```lua
import('android.content.Intent')
import('android.net.Uri')

local actOps = UtilsMain.activityOps()
local intent = Intent(Intent.ACTION_VIEW, Uri.parse("https://www.baidu.com"))
actOps.startActivity(intent)
```

#### homeActivity()

回到桌面。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 无 | - | - | 无参数 |

| 返回值类型 | 说明 |
|-----------|------|
| void | 无返回值 |

**示例：**
```lua
local actOps = UtilsMain.activityOps()
actOps.homeActivity()
```

### 2. Activity 存活与结束

#### isActivityAlive(activity: Activity)

判断传入的 Activity 是否存活。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| activity | Activity | 是 | 目标 Activity 实例 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 存活返回 true |

**示例：**
```lua
-- 假设通过其他方式拿到 activity 实例
local actOps = UtilsMain.activityOps()
print(actOps.isActivityAlive(activity))
```

#### finishActivity(activity: Activity)

结束指定 Activity。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| activity | Activity | 是 | 目标 Activity 实例 |

| 返回值类型 | 说明 |
|-----------|------|
| void | 无返回值 |

**示例：**
```lua
local actOps = UtilsMain.activityOps()
actOps.finishActivity(activity)
```



## OOP 调用例子


```lua
-- ============================================
-- Activity 操作示例
-- 演示如何使用 XfxPlugin.lua 调用 actOps 进行 Activity 管理
-- ============================================
import('com.nx.assist.lua.LuaEngine')

print('========== Activity 操作示例 ==========')

-- 加载 XfxPlugin.lua 类
local xfxModule = require('lib/XfxPlugin')

-- 创建 XFX 对象实例
local XFX = xfxModule:new({
    apkName = 'xfxPlugin-release.apk',  -- APK 文件名
})

-- 获取 actOps 对象
local actOps = XFX:getOps('activityOps')
if not actOps then
    error('无法获取 actOps 对象')
end

-- 设置上下文
local context = LuaEngine.getContext()
actOps.setContext(context)

print('开始 Activity 操作示例...')


-- ============================================
-- 2. 检查 Activity 是否存在
-- ============================================
print('\n--- 2. 检查 Activity 是否存在 ---')

-- 检查系统设置 Activity
local settingsExists = actOps.isActivityExists(
    'com.android.settings',
    'com.android.settings.Settings'
)
print('系统设置 Activity 是否存在: ' .. tostring(settingsExists))

-- 检查不存在的 Activity
local fakeExists = actOps.isActivityExists(
    'com.example.fake',
    'com.example.fake.FakeActivity'
)
print('不存在的 Activity 是否存在: ' .. tostring(fakeExists))

-- ============================================
-- 4. 返回桌面
-- ============================================
print('\n--- 4. 返回桌面 ---')

print('准备返回桌面...')
actOps.homeActivity()
print('已返回桌面')
sleep(1000)

-- ============================================
-- 5. 启动 Activity（使用 Intent）
-- ============================================
print('\n--- 5. 启动 Activity（使用 Intent） ---')

import('android.content.Intent')
import('android.content.ComponentName')

-- 方式一：通过包名和类名启动
local packageName = 'com.android.settings'
local className = 'com.android.settings.Settings'

if actOps.isActivityExists(packageName, className) then
    local intent = Intent()
    intent.setComponent(ComponentName(packageName, className))
    intent.setFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
    
    print('启动系统设置...')
    actOps.startActivity(intent)
    print('已启动系统设置')
    sleep(2000)
end

-- 方式二：通过 Action 启动
local intent2 = Intent(Intent.ACTION_VIEW)
intent2.setFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
-- 可以设置更多参数，如 URI、数据等
-- intent2.setData(android.net.Uri.parse('https://www.example.com'))

-- ============================================
-- 8. 综合示例：检查并启动应用
-- ============================================
print('\n--- 8. 综合示例：检查并启动应用 ---')

local targetPackage = 'com.android.settings'
local targetClass = 'com.android.settings.Settings'

-- 检查 Activity 是否存在
if actOps.isActivityExists(targetPackage, targetClass) then
    print('目标 Activity 存在，准备启动...')
    
    -- 创建 Intent
    local intent = Intent()
    intent.setComponent(ComponentName(targetPackage, targetClass))
    intent.setFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
    
    -- 启动 Activity
    actOps.startActivity(intent)
    print('已启动目标 Activity')
    
else
    print('目标 Activity 不存在')
end

print('\n========== Activity 操作示例完成 ==========')
```

## 注意事项

- **初始化要求**：使用前需先调用 `xxxOps.setContext(context)`或`UtilsMain.init(context)` 初始化上下文。
- **权限与上下文**：启动 Activity 时需在有合适上下文/任务栈的环境运行；某些场景需添加 `FLAG_ACTIVITY_NEW_TASK`。
