## 模块简介

`FloatLog` 除了提供悬浮日志窗口以外，还扩展了 Toast 弹窗。

有些系统的Toast，显示不正常。所以这里使用悬浮窗实现了模拟的Toast。

> ⚠️ 不支持懒人精灵`最高权限`下运行(最高权限下获取不到Context)。

## 功能概览

- **上下文设置**：设置 Context（setContext，与 UtilsMain.init 二选一）
- **Toast**：多重重载，支持位置、颜色、时长等自定义

## API 接口列表

#### setContext()

按需设置 Context（建议传入 applicationContext），在未调用 `UtilsMain.init` 时也可提前注入。

> setContext()和 UtilsMain.init() 二选一调用即可。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| context | Context | 是 | 应用或 Activity 的 context，建议 applicationContext |

| 返回值类型 | 说明 |
|-----------|------|
| void | 无 |

**示例：**
```lua
import('com.nx.assist.lua.LuaEngine')

local floatLogOps = UtilsMain.floatLog()

local context = LuaEngine.getContext()
floatLogOps.setContext(context)  -- 传入已有的 context
```

#### showToast()

异步显示 Toast 。


> 如果有多个 Toast 同时显示，则只显示最后一个。因为每次显示的时候，内部都会先销毁之前的，避免堆叠。

| 常用重载 | 关键参数 |
|----------|---------|
| showToast(message) | 文本 |
| showToast(message, duration=2000) | 文本、时长 |
| showToast(message, duration, gravityInt) | 文本、时长、位置 0/1/2 |
| showToast(message, duration, gravityInt, textColor, textSize, backgroundColor, padding=16, margin=50) | 全自定义 |




#### closeToast()

关闭当前 Toast。

| 返回值类型 | 说明 |
|-----------|------|
| void | 无 |

## 完整示例(4)

```lua
import('com.nx.assist.lua.LuaEngine')

local loader = LuaEngine.loadApk('xfxPlugin-release.apk')

local floatLogOps = UtilsMain.floatLog()

floatLogOps.setContext(context)


-- 方式一：最简单的调用（默认底部显示，2秒）
floatLogOps.showToast('这是一条 Toast 消息')

sleep(3000)

-- 方式二：自定义持续时间（显示3秒）
floatLogOps.showToast('显示3秒', 3000)

-- 方式三：使用整数指定位置（0=顶部, 1=中间, 2=底部）

floatLogOps.showToast('中间显示', 2000, 1)  -- 中间，2秒

sleep(3000)

floatLogOps.showToast('底部显示', 2000, 2)  -- 底部，2秒

sleep(3000)

-- 方式四：完全自定义样式（需要导入 Color）
import('android.graphics.Color')
floatLogOps.showToast(
    '自定义样式消息',
    3000,  -- 持续时间
    2,     -- 位置：2=底部
    Color.YELLOW,  -- 文字颜色
    16.0,  -- 文字大小
    0x99FF0000,  -- 背景颜色
    16,
    50
)

sleep(4000)

-- 方式五：手动关闭 Toast
floatLogOps.showToast('这条消息会被提前关闭', 10000)
sleep(2000)
floatLogOps.closeToast()  -- 手动关闭

print('Toast 测试完成')

```



## OOP 调用例子

使用 `XfxPlugin.lua` 封装的 Toast 接口，可以更方便地在懒人精灵中调用。

注意创建实例的时候，要传入上下文对象。

> 用户也可以根据自己的业务需求，封装独立的 Lua 模块。

### 基础用法

```lua
import('com.nx.assist.lua.LuaEngine')

-- 加载 XfxPlugin.lua 类
local xfxModule = require('lib/XfxPlugin')

-- 创建 XFX 对象实例
local XFX = xfxModule:new({
    apkName = 'xfxPlugin-release.apk',  -- APK 文件名
    context = LuaEngine.getContext(),  -- 上下文对象（不支持最高权限）
})

-- 方式一：最简单的调用（默认底部显示，2秒）
XFX:showToast('这是一条 Toast 消息')

sleep(3000)

-- 方式二：自定义持续时间
XFX:showToast('显示5秒', 30000)
sleep(5000)

XFX:closeToast()  -- 手动关闭

```

### 结合悬浮窗使用

```lua
import('com.nx.assist.lua.LuaEngine')

-- 加载 XfxPlugin.lua 类
local xfxModule = require('lib/XfxPlugin')

-- 创建 XFX 对象实例
local XFX = xfxModule:new({
    apkName = 'xfxPlugin-release.apk',  -- APK 文件名
    context = LuaEngine.getContext(),  -- 上下文对象（不支持最高权限）
})

-- 显示悬浮窗
-- XFX:showFloatLogWindow()  -- 也可使用默认参数
XFX:showFloatLogWindow({
    -- 配置悬浮窗
    left = 0,
    top = 200,
    width = 400,
    height = 300,
    isShowTimes = true,
    isShowTitle = true,
    maxCount = 5,
})

-- 设置标题
XFX:setFloatLogTitle('任务执行中...')

-- 使用 Toast 提示开始
XFX:showToast('任务开始')
sleep(3000)

-- 执行任务并显示进度
for i = 1, 10 do
    XFX:setFloatLogProgress(i / 10)
    XFX:setFloatLogInfo('进度：' .. (i * 10) .. '%')
    XFX:logi('处理第 ' .. i .. ' 项')
    
    -- 每完成一项显示 Toast
    if i % 3 == 0 then
        XFX:showToast('已完成 ' .. i .. ' 项', 1500)
    end
    
    sleep(1000)
end

-- 任务完成提示
XFX:showToast('任务完成！', 3000)
XFX:logs('所有任务已完成')

sleep(4000)
```



## 注意事项

- **初始化要求**：使用前需先调用 `xxxOps.setContext(context)`或`UtilsMain.init(context)` 初始化上下文。
- **权限要求**：需要悬浮窗权限才能显示 Toast
- **线程要求**：插件内部已约束在 UI 线程中调用。实际上主线程也可以调用，但 UI 线程更稳定
- **位置参数**：
   - `0` = 顶部 (TOP)
   - `1` = 中间 (CENTER)
   - `2` = 底部 (BOTTOM，默认)
- **持续时间**：单位为毫秒，默认 2000ms（2秒）
- **自动关闭**：Toast 会在指定时间后自动消失，也可以手动调用 `closeToast()` 关闭
