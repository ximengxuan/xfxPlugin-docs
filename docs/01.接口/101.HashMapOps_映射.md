## 模块简介

`HashMapOps` 提供线程安全的映射（Map）管理，支持按 ID 持有多个映射实例，覆盖创建、存取、移除、键值判断、状态查询、批量导入、清空/删除以及导出 JSON。

除默认的 **ConcurrentHashMap**（无序、高并发）外，还支持 **LinkedHashMap** 模式：按插入顺序遍历，并可设置最大缓存条数，超出后自动删除最早插入的数据，避免内存溢出。同时支持通过 **MMKV** 或 **SharedPreferences** 将数据持久化到本地。

典型场景：缓存账号与状态的对应关系、记录设备 ID 与配置参数的映射、带容量上限的有序缓存、脚本重启后恢复历史数据。

## 功能概览

- **实例管理**：创建/删除映射实例，支持 HashMap 与 LinkedHashMap 两种模式
- **容量控制**：LinkedHashMap 模式可设 `maxSize`，超出自动淘汰最早数据
- **持久化**：create 时自动持久化，或手动调用 save/load、saveToMmkv/saveToSp
- **键值操作**：存入、获取、移除、键/值包含判断
- **状态查询**：获取大小、最大条数、是否有序、判断是否为空
- **批量与导出**：批量存入（JSON 对象）、导出键/值/完整 JSON 对象

## API 接口列表

### 1. 实例管理

#### create()

创建或获取映射实例。支持两种调用方式：

**方式一：简单创建（ConcurrentHashMap，默认）**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 映射唯一标识 |

**方式二：高级创建（LinkedHashMap / 容量限制 / 持久化）**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 映射唯一标识 |
| ordered | Boolean | 是 | 是否使用 LinkedHashMap（按插入顺序遍历） |
| maxSize | Int | 是 | 最大条数，超出后删除最早数据；0 表示不限制 |
| persistType | String | 是 | 持久化类型：`none` / `mmkv` / `sp` |
| persistKey | String | 否 | 持久化存储键，整份映射存为 JSON 对象 |
| storageName | String | 否 | MMKV 实例 ID 或 SP 文件名，空串用默认实例 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 创建/获取成功返回 true |

> **说明**：若实例已存在，直接返回 true，不会覆盖已有配置。`maxSize > 0` 时会自动启用有序模式。启用持久化后，create 时自动从本地加载；put/remove/clear/putAll 后自动保存。

**示例（默认 HashMap）：**
```lua
local hashMapOps = UtilsMain.hashMapOps()

-- 无序、无条数限制、不持久化
print(hashMapOps.create("accountStatus"))
```

**示例（LinkedHashMap + 最大 100 条 + MMKV 持久化）：**
```lua
local hashMapOps = UtilsMain.hashMapOps()
hashMapOps.create("ipCache", true, 100, "mmkv", "ip_cache_data", "my_mmkv")
```

**示例（LinkedHashMap + 最大 50 条 + SP 持久化）：**
```lua
local hashMapOps = UtilsMain.hashMapOps()
hashMapOps.create("accountStatus", true, 50, "sp", "account_status_map", "xfx_hashmap")
```

#### remove()

删除 HashMap 实例（单参数重载，删除整个实例）。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 映射唯一标识 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 删除成功返回 true |

**示例：**
```lua
local hashMapOps = UtilsMain.hashMapOps()
print(hashMapOps.remove("accountStatus"))
```

### 2. 键值操作

#### put()

存入键值对。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 映射唯一标识 |
| key | String | 是 | 键 |
| value | String | 否 | 值，空则存入空串 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 键原本不存在并成功存入返回 true，键已存在（覆盖）返回 false |

**示例：**
```lua
local hashMapOps = UtilsMain.hashMapOps()
hashMapOps.create("accountStatus")
print(hashMapOps.put("accountStatus", "user001", "active"))
```

#### get()

根据键获取值。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 映射唯一标识 |
| key | String | 是 | 键 |

| 返回值类型 | 说明 |
|-----------|------|
| String? | 对应的值，键不存在返回 null |

**示例：**
```lua
local hashMapOps = UtilsMain.hashMapOps()
hashMapOps.create("accountStatus")
hashMapOps.put("accountStatus", "user001", "active")
print(hashMapOps.get("accountStatus", "user001"))
```

#### remove()

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|

| key | String | 是 | 键 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 键存在并成功移除返回 true |

**示例：**
```lua
local hashMapOps = UtilsMain.hashMapOps()
hashMapOps.create("accountStatus")
hashMapOps.put("accountStatus", "user001", "active")
print(hashMapOps.remove("accountStatus", "user001"))
```

### 3. 状态查询

#### size()

获取映射大小（键值对数量）。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 映射唯一标识 |

| 返回值类型 | 说明 |
|-----------|------|

**示例：**

#### isEmpty()

判断映射是否为空。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 映射唯一标识 |

| 返回值类型 | 说明 |
|-----------|------|

**示例：**
```lua
local hashMapOps = UtilsMain.hashMapOps()
```

#### getMaxSize()

获取映射配置的最大条数。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 映射唯一标识 |

| 返回值类型 | 说明 |
|-----------|------|

**示例：**


### 4. 持久化

#### save() / load()

使用 create 时配置的 persistType / persistKey 手动保存或加载。

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 映射唯一标识 |

| 返回值类型 | 说明 |
|-----------|------|
| Boolean | 成功返回 true，未配置持久化或失败返回 false |

**示例：**
```lua
local hashMapOps = UtilsMain.hashMapOps()
hashMapOps.create("accountStatus", true, 0, "mmkv", "account_map", "")
hashMapOps.put("accountStatus", "user001", "active")
print(hashMapOps.save("accountStatus"))
print(hashMapOps.load("accountStatus"))
```

#### saveToMmkv() / loadFromMmkv()

使用 `MMKVOps` 将映射保存到 MMKV 或从中加载（不依赖 create 的持久化配置）。

| 方法 | 参数 | 说明 |
|------|------|------|
| saveToMmkv(id, persistKey) | id: 映射 ID；persistKey: MMKV 键 | 保存整份 JSON |
| loadFromMmkv(id, persistKey) | 同上 | 加载并覆盖内存数据 |

**示例：**
```lua
local hashMapOps = UtilsMain.hashMapOps()
local mmkvOps = UtilsMain.mmkvOps()

hashMapOps.create("accountStatus")
hashMapOps.put("accountStatus", "user001", "active")
hashMapOps.saveToMmkv("accountStatus", "manual_account_map")

-- 重启脚本后恢复
hashMapOps.create("accountStatus")
hashMapOps.loadFromMmkv("accountStatus", "manual_account_map")
print(hashMapOps.get("accountStatus", "user001"))
```

#### saveToSp() / loadFromSp()

使用 `SPStaticOps` 将映射保存到 SharedPreferences 或从中加载。

| 方法 | 参数 | 说明 |
|------|------|------|
| saveToSp(id, persistKey, spName) | spName 可为空串（默认 SP） | 保存整份 JSON |
| loadFromSp(id, persistKey, spName) | 同上 | 加载并覆盖内存数据 |

**示例：**
```lua
local hashMapOps = UtilsMain.hashMapOps()

hashMapOps.create("accountStatus")
hashMapOps.put("accountStatus", "user001", "active")
hashMapOps.saveToSp("accountStatus", "account_map", "xfx_hashmap")

-- 重启脚本后恢复
hashMapOps.create("accountStatus")
hashMapOps.loadFromSp("accountStatus", "account_map", "xfx_hashmap")
print(hashMapOps.get("accountStatus", "user001"))
```

## 完整示例

```lua
import('com.nx.assist.lua.LuaEngine')
local loader = LuaEngine.loadApk('xfxPlugin-release.apk'); assert(loader)
local context = LuaEngine.getContext()
local UtilsMain = loader.loadClass('com.xfx.plugin.UtilsMain'); assert(UtilsMain)
UtilsMain.init(context)

local hashMapOps = UtilsMain.hashMapOps()

-- 创建
print("create:", hashMapOps.create("accountStatus"))

-- 存入
print("put1:", hashMapOps.put("accountStatus", "user001", "active"))
print("put2:", hashMapOps.put("accountStatus", "user002", "frozen"))
print("put overwrite:", hashMapOps.put("accountStatus", "user001", "inactive"))  -- false

-- 获取
print("get:", hashMapOps.get("accountStatus", "user001"))

-- 包含判断
print("containsKey:", hashMapOps.containsKey("accountStatus", "user002"))
print("containsValue:", hashMapOps.containsValue("accountStatus", "frozen"))

-- 状态
print("size:", hashMapOps.size("accountStatus"))
print("isEmpty:", hashMapOps.isEmpty("accountStatus"))

-- 批量存入
print("putAll:", hashMapOps.putAll("accountStatus", '{"user003":"active","user004":"active"}'))

-- 导出
print("keys:", hashMapOps.keys("accountStatus"))
print("values:", hashMapOps.values("accountStatus"))
print("json:", hashMapOps.toJsonObject("accountStatus"))

-- 移除键值对
print("remove key:", hashMapOps.remove("accountStatus", "user002"))

-- 清空
print("clear:", hashMapOps.clear("accountStatus"))

-- 删除实例
print("remove instance:", hashMapOps.remove("accountStatus"))
```

## OOP 示例（账号状态缓存）

```lua
local xfxModule = require('lib/XfxPlugin')
local XFX = xfxModule:new({ apkName = 'xfxPlugin-release.apk' })

local hashMapOps = XFX:getOps('hashMapOps')

local MAP_ID = 'accountStatus'
hashMapOps.create(MAP_ID)

local function setAccountStatus(account, status)
    hashMapOps.put(MAP_ID, account, status)
end

local function getAccountStatus(account)
    return hashMapOps.get(MAP_ID, account)
end

setAccountStatus('user001', 'active')
setAccountStatus('user002', 'frozen')

if getAccountStatus('user002') == 'frozen' then
    print('账号 user002 已被冻结，跳过任务')
end

print('全部账号：' .. hashMapOps.toJsonObject(MAP_ID))
```

## LinkedHashMap 有序缓存 + 最大条数限制

```lua
import('com.nx.assist.lua.LuaEngine')
local loader = LuaEngine.loadApk('xfxPlugin-release.apk'); assert(loader)
local UtilsMain = loader.loadClass('com.xfx.plugin.UtilsMain'); assert(UtilsMain)
UtilsMain.init(LuaEngine.getContext())

local hashMapOps = UtilsMain.hashMapOps()
local MAP_ID = 'ipCache'

hashMapOps.put(MAP_ID, 'ip1', '192.168.1.1')
hashMapOps.put(MAP_ID, 'ip2', '192.168.1.2')
hashMapOps.put(MAP_ID, 'ip3', '192.168.1.3')

print('keys:', hashMapOps.keys(MAP_ID))           -- ["ip1","ip2","ip3"]

-- 第 4 条插入后，最早的 ip1 被自动删除
hashMapOps.put(MAP_ID, 'ip4', '10.0.0.1')
print('size:', hashMapOps.size(MAP_ID))           -- 3
print('keys:', hashMapOps.keys(MAP_ID))           -- ["ip2","ip3","ip4"]
print('contains ip1:', hashMapOps.containsKey(MAP_ID, 'ip1'))  -- false
```

## 手动使用 mmkvOps / SPStaticOps 持久化

适合需要自行控制保存时机，或将映射数据与其他 MMKV/SP 数据统一管理的场景。

```lua
import('com.nx.assist.lua.LuaEngine')
local loader = LuaEngine.loadApk('xfxPlugin-release.apk'); assert(loader)
local UtilsMain = loader.loadClass('com.xfx.plugin.UtilsMain'); assert(UtilsMain)
UtilsMain.init(LuaEngine.getContext())

local hashMapOps = UtilsMain.hashMapOps()
local mmkvOps = UtilsMain.mmkvOps()
local spStaticOps = UtilsMain.spStaticOps()

local MAP_ID = 'accountStatus'
hashMapOps.create(MAP_ID)

-- 写入数据
hashMapOps.put(MAP_ID, 'user001', 'active')
hashMapOps.put(MAP_ID, 'user002', 'frozen')

-- 方式一：通过 HashMapOps 封装方法
hashMapOps.saveToMmkv(MAP_ID, 'manual_map_key')
hashMapOps.loadFromMmkv(MAP_ID, 'manual_map_key')

-- 方式二：直接使用 mmkvOps（等价操作）
local json = hashMapOps.toJsonObject(MAP_ID)
mmkvOps.put('manual_map_key', json)
local loaded = mmkvOps.get('manual_map_key', '{}')
hashMapOps.clear(MAP_ID)
hashMapOps.putAll(MAP_ID, loaded)

-- 方式三：直接使用 SPStaticOps
spStaticOps.setInstance('xfx_hashmap')
spStaticOps.put('manual_map_key', hashMapOps.toJsonObject(MAP_ID))
local spJson = spStaticOps.get('manual_map_key', '{}')
hashMapOps.clear(MAP_ID)
hashMapOps.putAll(MAP_ID, spJson)
```

## HashMap 与 LinkedHashMap 对比

| 对比维度 | HashMap（默认 create） | LinkedHashMap（ordered=true） |
| ---- | ---- | ---- |
| **底层实现** | ConcurrentHashMap | LinkedHashMap + synchronized |
| **遍历顺序** | 不保证 | 按插入顺序 |
| **最大条数** | 不支持 | 支持 maxSize，超出删最早 |
| **并发性能** | 更高 | 略低（单锁同步） |
| **典型用途** | 通用键值缓存 | 有序缓存、容量受限缓存 |

## 注意事项

- **ID 一致性**：所有操作需使用同一映射 ID；未创建的 ID 将导致操作失败。
- **键值类型**：键和值均为 String 类型；复杂数据请序列化为 JSON 字符串后存入。
- **remove 重载**：`remove(id)` 删除整个实例；`remove(id, key)` 移除单个键值对。
- **put 返回值**：返回 true 表示键是新插入的，false 表示键已存在并被覆盖。
- **maxSize 限制**：仅有序模式生效；超出后按**插入顺序**删除最早的数据（非 LRU 访问顺序）。
- **持久化格式**：整份映射以 JSON 对象字符串存储在单个 persistKey 下；启用自动持久化后，每次 put/remove/clear 都会写入本地。
- **create 幂等**：同一 ID 重复 create 不会更新配置，需先 `remove(id)` 再重新 create。
- **并发安全**：HashMap 模式使用 ConcurrentHashMap；LinkedHashMap 模式使用 synchronized，多线程场景建议业务层加锁。
- **生命周期**：脚本长期运行时建议在不使用时调用 `remove(id)` 释放实例，避免内存占用。
